### 16:45 - Research: Why Claude Code Lacks Memory

**Created**: `outputs/.../why_claude_code_lacks_memory_research.md`

Key findings:
1. **Stateless by design** - Anthropic API is intentionally stateless
2. **Transparency over automation** - Explicit memory (CLAUDE.md) preferred over black-box RAG
3. **Technical challenges** - Memory retrieval is hard (73% lag vs humans in temporal reasoning)
4. **Privacy/security** - No unexpected recall of sensitive info
5. **Community demand exists** - Multiple GitHub issues requesting this feature
6. **Official solution is CLAUDE.md + Hooks** - Not automatic, but intentional

Sources: GitHub issues #14227, #2954, #12646, #18417; Claude API docs; LoCoMo research; Mem0 paper
