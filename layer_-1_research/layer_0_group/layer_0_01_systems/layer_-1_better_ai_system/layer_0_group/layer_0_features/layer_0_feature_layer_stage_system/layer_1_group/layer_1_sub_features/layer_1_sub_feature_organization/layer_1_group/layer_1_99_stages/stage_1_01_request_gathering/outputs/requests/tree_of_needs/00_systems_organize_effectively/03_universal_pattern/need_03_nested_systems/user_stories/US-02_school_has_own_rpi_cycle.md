# School System Has Own R/P/I Cycle

**As a** school system nested within the broader AI system,
**I want** my own independent research/production/instantiation cycle,
**So that** educational features can be researched, stabilized, and personalized independently of the parent AI system's R/P/I cycle.

## Acceptance Criteria

**Scenario 1: School system has its own research entities**
- **Given** the school system wants to experiment with a new teaching approach,
- **When** a research entity is created for this experiment,
- **Then** it lives within the school system's own `layer_-1_research/` (not the parent AI system's research layer) and follows the same R/P/I pattern recursively.

**Scenario 2: School system has its own production patterns**
- **Given** the school system has validated a teaching methodology through research,
- **When** it is promoted to production within the school system,
- **Then** it becomes part of the school system's production template — available to all student instances — independent of whether the parent AI system has adopted or even knows about this methodology.

**Scenario 3: School system's R/P/I cycle mirrors the parent pattern**
- **Given** I compare the school system's R/P/I structure with the parent AI system's structure,
- **When** I examine the directory layout, promotion workflow, and instantiation process,
- **Then** they follow the identical pattern — research entities, promotion protocol, production templates, instance creation — demonstrating that the R/P/I pattern is truly recursive and self-similar at every level.
