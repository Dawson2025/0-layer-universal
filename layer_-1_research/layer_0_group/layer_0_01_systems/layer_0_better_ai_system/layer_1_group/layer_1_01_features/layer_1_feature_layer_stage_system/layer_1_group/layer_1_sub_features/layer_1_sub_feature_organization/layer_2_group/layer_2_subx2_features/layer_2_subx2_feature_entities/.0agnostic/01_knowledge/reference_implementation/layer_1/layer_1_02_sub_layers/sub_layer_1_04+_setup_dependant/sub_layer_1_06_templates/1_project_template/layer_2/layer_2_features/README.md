# Features

This folder contains features (Layer 2) within this project.

Each feature follows the standard feature template structure (`2_feature_template`).

## Naming Convention

```
layer_2_feature_<name>/
```

## When to Create a Feature

Create a feature when you have:
- A distinct topic or functional area within the project
- Content that benefits from organized study/development stages
- Topics that need further breakdown into sub-features or components

## Note on "Sub" Prefix

The "sub" prefix indicates nesting within the **same type**:
- Feature inside a project = **feature** (different types, NO "sub")
- Feature inside a feature = **sub_feature** (same type nesting)
- Project inside a project = **sub_project**

Since this is a feature inside a project (different types), it does NOT use the "sub" prefix.
