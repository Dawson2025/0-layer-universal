# Developer Reads One Guide

**As a** developer encountering the R/P/I pattern for the first time,
**I want to** read a single, self-contained guide that explains the pattern,
**So that** I understand the organizational structure without needing to read dozens of files scattered across the repository.

<!-- section_id: "5d9938ed-3f3c-48c0-8e35-2efcafbcca1a" -->
## Acceptance Criteria

**Scenario 1: Single document covers the core pattern**
- **Given** I open the R/P/I pattern guide,
- **When** I read it from start to finish,
- **Then** I understand: what research/production/instantiation means, how the three layers relate, the promotion workflow between them, and how instances inherit from production — all in one document.

**Scenario 2: Guide is self-contained**
- **Given** I have never seen the layer-stage system before,
- **When** I read the R/P/I pattern guide without any prior context,
- **Then** I can understand the pattern fully — the guide defines all necessary terms, explains all concepts, and does not require me to read prerequisite documents.

**Scenario 3: Guide includes a visual diagram and concrete example**
- **Given** I am a visual learner looking at the guide,
- **When** I scan for visual aids,
- **Then** I find at least one diagram showing the R/P/I structure (research → promotion → production → instantiation) and at least one concrete example (e.g., the school system) that shows how the pattern applies to a real domain.
