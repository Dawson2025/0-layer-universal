# REQ-03: Self-Healing Context Files

**Need**: [UUID-Based Reference Resolution](../README.md)

<!-- section_id: "5c6d7e8f-9a0b-4c1d-2e3f-4a5b6c7d8e9f" -->
## Requirements

- **MUST** emit UUID references with `resolve-uuid` instructions in all generated context files (CLAUDE.md, AGENTS.md, GEMINI.md, OPENAI.md, .cursorrules, copilot-instructions.md)
- **MUST** include a resolve-uuid usage section in each generated file so AI apps know how to resolve references
- **MUST** ensure references resolve correctly even if `agnostic-sync.sh` has not been re-run since the last move (self-healing — no stale window)
- **MUST** work with all major AI coding apps: Claude Code, Codex CLI, Gemini CLI, Copilot (VS Code + CLI), Cursor IDE agent, Windsurf, Aider, Cline, Continue.dev
- **SHOULD** include a human-readable path alongside the UUID reference for developer convenience (but the UUID is the stable reference)
- **SHOULD** provide a fallback for any future AI apps that cannot run bash: pre-resolved paths generated by `agnostic-sync.sh`
- **MUST NOT** require regeneration of context files for references to remain valid after a move — only the UUID index needs rebuilding
- **MUST NOT** increase context file size by more than 20% compared to current path-only references
