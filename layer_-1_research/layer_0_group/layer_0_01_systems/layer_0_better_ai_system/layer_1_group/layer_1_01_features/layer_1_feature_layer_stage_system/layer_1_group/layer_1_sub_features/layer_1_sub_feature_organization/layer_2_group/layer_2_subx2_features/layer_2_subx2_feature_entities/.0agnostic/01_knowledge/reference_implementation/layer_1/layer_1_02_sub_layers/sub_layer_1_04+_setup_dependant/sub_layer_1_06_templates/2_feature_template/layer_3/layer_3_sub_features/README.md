# Sub-Features

This folder contains sub-features (Layer 3) within this feature.

Each sub-feature follows the standard feature template structure (`2_feature_template`).

## Naming Convention

```
layer_3_subx3_feature_<name>/
```

## When to Create a Sub-Feature

Create a sub-feature when you have:
- A distinct topic or functional area within the parent feature
- Content that benefits from organized study/development stages
- Topics that need further breakdown

## Note on "Sub" Prefix

The "sub" prefix indicates nesting within the **same type**:
- Feature inside a project = **feature**
- Feature inside a feature = **sub_feature** (same type nesting)
- Project inside a project = **sub_project**

Since this is a feature inside another feature (same types), it uses the "sub" prefix.
