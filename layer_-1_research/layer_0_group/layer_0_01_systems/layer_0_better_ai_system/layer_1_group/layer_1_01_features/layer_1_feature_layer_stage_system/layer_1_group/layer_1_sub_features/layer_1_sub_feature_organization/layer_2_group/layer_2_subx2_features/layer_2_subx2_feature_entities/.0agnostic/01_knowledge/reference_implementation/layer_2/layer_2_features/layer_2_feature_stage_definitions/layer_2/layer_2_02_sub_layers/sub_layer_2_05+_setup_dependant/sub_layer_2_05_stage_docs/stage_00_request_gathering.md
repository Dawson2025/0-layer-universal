# Stage 00: Request Gathering

<!-- section_id: "92ec3f04-e5ec-4029-8f30-feebb9cadd17" -->
## Purpose
Collect, clarify, and document initial requirements from stakeholders. This stage ensures all necessary information is gathered before work begins, preventing scope creep and misunderstandings later in the workflow.

<!-- section_id: "56b41e39-b2a5-4857-bc0b-e88bbc7721bc" -->
## Entry Criteria
- A new project, feature, or task has been identified
- Stakeholder(s) available for requirements discussion
- Access to relevant context and background information

<!-- section_id: "a937ff7c-593b-4fe4-b34c-8ca064995a04" -->
## Exit Criteria
- All requirements documented in structured format
- Ambiguities resolved through stakeholder clarification
- Scope boundaries clearly defined
- Acceptance criteria established
- Requirements reviewed and approved by stakeholder

<!-- section_id: "08bbdc73-1900-4869-b6ae-52ed9b995938" -->
## Typical Tasks
- Conduct stakeholder interviews
- Document functional requirements
- Document non-functional requirements
- Identify constraints and dependencies
- Define scope boundaries
- Establish acceptance criteria
- Create requirements traceability matrix
- Prioritize requirements (must-have vs nice-to-have)

<!-- section_id: "597cca9f-91ff-4db1-84ee-f1db89194593" -->
## Handoffs
- **From Previous**: Initial request or project trigger
- **To Next**: Completed requirements document, scope definition, acceptance criteria

## Key Artifacts
- Requirements specification document
- Stakeholder interview notes
- Scope definition document
- Acceptance criteria checklist
- Constraints and assumptions list

<!-- section_id: "e0d3a660-ab15-4645-b2bc-d18d5d45ae42" -->
## Common Pitfalls
- Insufficient stakeholder engagement
- Assuming requirements instead of asking
- Scope too vague or too broad
- Missing non-functional requirements
- Not documenting constraints
