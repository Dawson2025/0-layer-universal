# Features

This folder contains features (Layer 3) within this sub-project.

Each feature follows the standard feature template structure (`2_feature_template`).

<!-- section_id: "7286c8bd-3556-42b3-839e-9fb7da91d677" -->
## Naming Convention

```
layer_3_feature_<name>/
```

<!-- section_id: "9b2b21b5-f4ba-4282-be9f-38e507189177" -->
## When to Create a Feature

Create a feature when you have:
- A distinct topic or functional area within the sub-project
- Content that benefits from organized study/development stages
- Topics that need further breakdown into sub-features or components

<!-- section_id: "125c1d03-bade-42a3-b30b-efc7a1a8de2c" -->
## Note on "Sub" Prefix

The "sub" prefix indicates nesting within the **same type**:
- Feature inside a project = **feature**
- Feature inside a feature = **subfeature**
- Project inside a project = **subproject**

Since this is a feature inside a sub-project (different types), it's just called a "feature", not a "subfeature".
