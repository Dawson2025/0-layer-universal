# Need: Promotion Workflow

**Branch**: [Research/Production Lifecycle](../README.md)

<!-- section_id: "1bd3c84d-e85e-4216-908b-6c3df9d9f334" -->
## Definition

A system MUST have a clear protocol for promoting validated research findings to production status.

<!-- section_id: "97b2c910-2145-4b06-a8ae-434fd3d38eee" -->
## Why This Matters

Without a promotion workflow, research stays isolated forever (wasted effort) or gets copied ad-hoc to production (risky). The promotion workflow is the controlled bridge between experimental and stable.

<!-- section_id: "e27f212a-735a-4c74-86e9-9ea3e87b66c7" -->
## Acceptance Criteria

- [ ] Promotion protocol defines validation steps before promotion
- [ ] Promoted content is referenced (not copied) when possible
- [ ] Promotion is tracked (what was promoted, when, from where)
- [ ] Rollback is possible if promoted content causes issues
