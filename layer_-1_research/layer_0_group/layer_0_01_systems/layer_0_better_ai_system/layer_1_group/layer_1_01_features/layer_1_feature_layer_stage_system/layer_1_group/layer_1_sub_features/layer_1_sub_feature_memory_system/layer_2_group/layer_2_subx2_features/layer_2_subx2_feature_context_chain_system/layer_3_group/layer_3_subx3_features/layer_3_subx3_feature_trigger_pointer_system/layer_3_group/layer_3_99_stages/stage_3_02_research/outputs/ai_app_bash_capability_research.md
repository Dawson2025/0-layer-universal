# AI App Bash Capability Research

## Research Question

Which AI coding apps can execute bash/terminal commands as part of their workflow? This determines whether UUID-based references (requiring runtime resolution via a shell function) can replace hardcoded paths in their context files.

<!-- section_id: "d4e5f6a7-b8c9-4012-3def-012345678901" -->
## Why This Matters

The UUID identity system assigns stable identifiers to every entity, file, directory, section, and stage. To eliminate the "bajillion file changes" problem when things move, references need to use UUIDs instead of paths. But UUID resolution requires running a shell command:

```bash
resolve-uuid 08a4e9bc-8cc1-457e-b966-0a912ae6dff7
# Returns: .0agnostic/03_protocols/pointer_sync_protocol/tools/pointer-sync.sh
```

If an AI app can run bash, its context file (CLAUDE.md, AGENTS.md, etc.) can contain UUID references with `resolve-uuid` instructions. The app resolves at the moment of use — no stale paths, no regeneration needed.

If an AI app cannot run bash, its context file must contain pre-resolved paths generated by `agnostic-sync.sh`. These go stale when things move and require regeneration.

### Key Finding

**Every major AI coding app that can read filesystem-based context files can also run bash commands.** There are no mainstream AI coding apps that can read CLAUDE.md-style context but cannot execute terminal commands.

This means:
- UUID references with `resolve-uuid` instructions can go in **all** ported context files
- Pre-resolved paths are a **theoretical fallback** for future apps that might lack bash, not a current practical need
- The "split by capability" design simplifies to: UUID references everywhere, with the resolution mechanism documented in each file format

## Cursor Deep Dive

### Cursor IDE Agent

- **Can run bash**: Yes
- **How**: Agent mode executes terminal commands in a sandboxed environment within the IDE
- **YOLO mode**: Auto-approves terminal commands without user confirmation
- **Limitations**: Commands run through the IDE's terminal integration, not raw shell. Some commands may require user approval without YOLO mode enabled.
- **Context file**: `.cursorrules` — can include UUID + resolve-uuid instructions since the agent can execute bash

### Cursor CLI

- **Status**: Does not exist as a product (as of March 2026)
- **Cursor's CLI story**: Cursor offers `cursor` as a command to open the IDE (similar to `code` for VS Code), but there is no standalone terminal-based AI agent like Claude Code or Gemini CLI
- **Implication**: No `.cursorrules`-equivalent CLI context file needed. The IDE agent's `.cursorrules` is sufficient.

## Implications for UUID Resolution System

1. **All context files can use UUID references**: Since every app can run bash, `agnostic-sync.sh` can emit UUID + resolve-uuid patterns in all generated files (CLAUDE.md, AGENTS.md, GEMINI.md, OPENAI.md, .cursorrules, copilot-instructions.md)

2. **No app needs pre-resolved paths as primary mechanism**: Pre-resolved paths can be included as human-readable comments alongside UUID references, but they're not required for any current app

3. **Resolution function must be lightweight**: Since it runs on every reference lookup, it needs to be fast (<10ms). The current `.entity-lookup.tsv` grep approach (~5ms) meets this requirement.

4. **Sandboxing considerations**: Some apps (Codex CLI, Cursor) sandbox bash execution. The `resolve-uuid` function must work within sandboxed environments — it only reads a local TSV/JSON file, so no network or elevated permissions needed.
