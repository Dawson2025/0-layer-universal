# Need: General Pattern

**Branch**: [Universal Pattern](../README.md)

## Definition

The R/P/I (Research/Production/Instantiation) pattern MUST be documented as a domain-agnostic organizational structure that any system can adopt.

## Why This Matters

If the pattern only works for AI, it's limited. Documenting it as a general pattern makes it reusable across domains — software, education, organizational management, product development.

## Acceptance Criteria

- [ ] Pattern documented without domain-specific assumptions
- [ ] Reusable templates exist for creating R/P/I systems
- [ ] Pattern can be explained in one page to someone unfamiliar with the layer-stage system
- [ ] At least one non-AI example demonstrates the pattern
