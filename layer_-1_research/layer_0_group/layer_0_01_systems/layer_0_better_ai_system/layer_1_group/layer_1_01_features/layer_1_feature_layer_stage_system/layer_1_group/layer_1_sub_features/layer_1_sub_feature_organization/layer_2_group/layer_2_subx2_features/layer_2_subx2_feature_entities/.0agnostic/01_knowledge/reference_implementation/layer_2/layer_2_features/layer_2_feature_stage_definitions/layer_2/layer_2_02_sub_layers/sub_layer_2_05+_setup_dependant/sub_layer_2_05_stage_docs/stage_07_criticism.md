# Stage 06: Criticism

<!-- section_id: "4771f92a-2d7f-4a36-b217-57a238b461e1" -->
## Purpose
Conduct thorough review and critique of the tested product to identify improvements, issues, and areas that need refinement. This stage provides constructive feedback that drives quality improvement before final delivery.

<!-- section_id: "1116080c-1c67-42f3-ad1d-2c055db15bc6" -->
## Entry Criteria
- Testing completed from Stage 05
- Test results documented
- Product in stable, reviewable state
- Reviewers identified and available
- Review criteria established

<!-- section_id: "60881a64-ea60-4001-9e7c-1f2e572ddadf" -->
## Exit Criteria
- All review sessions completed
- Feedback documented and categorized
- Issues prioritized by severity
- Improvement recommendations provided
- Action items assigned
- Review summary approved

<!-- section_id: "eb9ec12a-e700-427b-8936-ea3a9f7032ea" -->
## Typical Tasks
- Conduct code reviews
- Perform design reviews
- Review against requirements
- Assess quality metrics
- Evaluate user experience
- Check documentation completeness
- Identify technical debt
- Assess maintainability
- Review security posture
- Gather stakeholder feedback

<!-- section_id: "54ac052a-7e2d-4f1f-a45a-26a38c525a06" -->
## Handoffs
- **From Previous**: Test results, defect reports, tested product
- **To Next**: Review findings, prioritized issues, improvement recommendations, action items

<!-- section_id: "c47b0bfc-1d57-43eb-9456-bca6816d2a9b" -->
## Common Pitfalls
- Not being constructive
- Focusing only on negatives
- Missing systemic issues
- Not prioritizing findings
- Vague feedback
- Skipping documentation review
