# GAB/AALang Agent System

## Summary

GAB (Generalized Agent Blueprint) is the AALang specification for defining AI agents as JSON-LD graphs. Each entity in the layer-stage system has a `.gab.jsonld` orchestrator that defines its modes, actors, personas, and constraints. These agent definitions are a key context delivery avenue (A5) -- agents read them to understand their operational modes and behavioral constraints.

Two primary patterns are used: the 4-mode-13-actor pattern for focused single-purpose agents (like the context loading agent) and the 5-mode-15-actor pattern for orchestrators that coordinate other agents. Each mode has Senior/Junior persona pairs that deliberate and verify each other's work. State actors persist across modes within a session. Our system extends AALang with cross-session persistence via file-based state.

Every .gab.jsonld has a matching `.integration.md` summary (auto-generated by `tools/jsonld-to-md.sh`). The integration summary provides the same information in readable markdown (~40 lines, ~200 tokens) compared to the full JSON-LD (~800 lines, ~2000 tokens). For most tasks, the integration summary is sufficient; only query the full .gab.jsonld when precise mode constraints or actor definitions are needed.

## Key Concepts

- **GAB = JSON-LD agent definitions** with modes, actors, personas, isolated states
- **Two patterns**: 4-mode-13-actor (focused agents) and 5-mode-15-actor (orchestrators)
- **Senior/Junior deliberation**: Each mode has a persona pair that verifies work
- **Integration summaries**: .integration.md as a lightweight cached summary of .gab.jsonld
- **Cross-session persistence**: Our extension -- state saved to file between sessions
