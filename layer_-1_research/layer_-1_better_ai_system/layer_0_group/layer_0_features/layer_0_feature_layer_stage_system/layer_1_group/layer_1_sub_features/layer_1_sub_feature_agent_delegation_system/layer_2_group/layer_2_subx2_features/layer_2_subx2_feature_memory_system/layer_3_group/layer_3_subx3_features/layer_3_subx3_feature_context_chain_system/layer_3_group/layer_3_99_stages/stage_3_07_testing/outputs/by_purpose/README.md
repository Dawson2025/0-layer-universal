# Testing Output Taxonomy (Canonical)

All new testing artifacts must be organized as:

`outputs/by_purpose/<purpose>/{design,implementation,runs,results,insights}/`

## Current Purposes
- `context_chain_validation`
- `codex_runtime_validation`
- `reports_funnel_validation`
- `avenue_web_validation`
- `full_suite_validation`

## Compatibility
- `outputs/by_topic/` remains available for legacy links only.
- New files should not be authored directly in `outputs/by_topic/`.
