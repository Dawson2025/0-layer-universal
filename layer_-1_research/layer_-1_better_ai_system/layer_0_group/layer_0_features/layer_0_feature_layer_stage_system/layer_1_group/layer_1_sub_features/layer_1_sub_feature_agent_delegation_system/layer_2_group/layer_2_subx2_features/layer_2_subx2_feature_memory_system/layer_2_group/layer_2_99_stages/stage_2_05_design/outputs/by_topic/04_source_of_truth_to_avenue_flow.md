# Source of Truth to Avenue Flow

## Overview

This document defines the holistic context ordering in the layer-stage system: how content flows from source of truth to delivery avenues, and how the sync pipeline connects them.

## Numbering Encodes the Flow

The `.0agnostic/` numbering is not arbitrary — it encodes the information flow:

| Number | Name | Role in Flow |
|--------|------|--------------|
| 01 | knowledge/ | Content: what we know |
| 02 | rules/ | Content: what we must do |
| 03 | protocols/ | Content: how we do things |
| 04 | episodic_memory/ | Content: what happened |
| 05 | handoff_documents/ | Content: what to pass on |
| 06 | context_avenue_web/ | **Delivery**: how content reaches consumers |
| 07+ | setup_dependant/ | **Environment**: OS/tool configuration |

01-05 are **producers**. They create and store content. They are edited by humans and agents.

06 is the **delivery mechanism**. It transforms and indexes content for consumption. It is generated by sync scripts.

07+ is the **environment layer**. It configures how the system runs on specific platforms.

## Avenue Web: Complete Ordering

### File-Based Avenues (01-08)

Ordered by comprehensiveness — how complete a picture each avenue provides:

| # | Avenue | What It Delivers | Comprehensiveness |
|---|--------|------------------|-------------------|
| 01 | AALang (.gab.jsonld) | Full agent graphs with modes, actors, personas, constraints | Most comprehensive — complete agent identity |
| 02 | AALang Markdown Integration | Markdown summaries of agent definitions | High — human-readable agent overview |
| 03 | Auto Memory | Persistent learnings across sessions | Medium — accumulated operational insights |
| 04 | @import References | Curated cross-file reference collections | Medium — selective pointers |
| 05 | Skills | Procedural SKILL.md files with WHEN/WHEN NOT | Medium — task-specific procedures |
| 06 | Agents | Lightweight .agent.jsonld stubs | Medium-Low — identity without full graph |
| 07 | Path-Specific Rules | Directory-scoped rules (.claude/rules/) | Low — narrow behavioral constraints |
| 08 | Hooks | Event-triggered shell scripts | Lowest — single-event responses |

### Data-Based Avenues (09-13)

Ordered by comprehensiveness — how complete a data model each avenue maintains:

| # | Avenue | What It Delivers | Comprehensiveness |
|---|--------|------------------|-------------------|
| 09 | Knowledge Graph | Structural model with typed relationships | Most comprehensive — full system topology |
| 10 | Relational Index | Tabular metadata across all entities | High — broad but flat |
| 11 | Vector Embeddings | Semantic similarity fingerprints | Medium — broad but lossy |
| 12 | Temporal Index | Time-series of events and changes | Low — single dimension (time) |
| 13 | SHIMI Structures | Per-node optimization primitives | Lowest — individual artifacts |

## Context Loading Order (Agent Perspective)

When an AI agent enters an entity, it loads context in this order:

1. **0AGNOSTIC.md STATIC** — always loaded, identifies the entity
2. **CLAUDE.md / tool file** — tool-specific version (auto-generated from step 1)
3. **Skills check** — match WHEN conditions against current task
4. **Rules check** — load applicable static rules, check dynamic triggers
5. **0AGNOSTIC.md DYNAMIC** — loaded on-demand when agent needs details
6. **.0agnostic/ resources** — read specific knowledge, protocols, episodic as needed
7. **Data-based avenues** — query for semantic search, status dashboards, temporal context (if available)

Steps 1-6 work with zero dependencies (file-based only). Step 7 is an optional accelerator.

## Key Architectural Properties

**Single direction of authority**: Source of truth (01-05) → Avenues (06). Never the reverse.

**Graceful degradation**: If databases are down, file-based avenues still work. Zero-dependency operation is always preserved.

**Unified sync trigger**: A single `sync-main.sh` can propagate a 0AGNOSTIC.md edit through all file-based AND data-based avenues.

**Consistent ordering convention**: Both file-based and data-based avenues follow the same comprehensiveness principle, making the system predictable and navigable.
