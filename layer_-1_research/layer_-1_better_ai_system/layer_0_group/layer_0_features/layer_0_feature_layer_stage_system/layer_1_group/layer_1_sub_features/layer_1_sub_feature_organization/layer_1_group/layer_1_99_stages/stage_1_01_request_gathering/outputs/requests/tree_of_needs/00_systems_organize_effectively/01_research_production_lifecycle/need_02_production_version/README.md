# Need: Production Version

**Branch**: [Research/Production Lifecycle](../README.md)

## Definition

A system MUST have a production version that contains stable, tried-and-true patterns that agents and users rely on by default.

## Why This Matters

Production is the default. When an agent starts work, it should encounter proven patterns without needing to know about ongoing research. Production provides the baseline that everyone trusts.

## Acceptance Criteria

- [ ] Production entities represent the standard entity structure (not tagged as research)
- [ ] Production content has been validated through the promotion workflow
- [ ] Production is the default mode — agents use production unless explicitly told otherwise
- [ ] Production includes version tracking for promoted content
