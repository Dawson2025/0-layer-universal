# layer_2_feature_stage_definitions

## Purpose
Defines the 11 workflow stages used throughout the Layer-Stage Framework.

## Stages
1. **00_request_gathering** - Gather initial requirements
2. **01_research** - Explore problem space, gather information
3. **02_instructions** - Define what needs to be done
4. **03_planning** - Plan how to do it
5. **04_design** - Design the solution
6. **05_development** - Implement the solution
7. **06_testing** - Test the implementation
8. **07_criticism** - Review and critique
9. **08_fixing** - Fix identified issues
10. **09_current_product** - Current working version
11. **10_archives** - Historical records

## Stage Structure
Each stage contains:
- `ai_agent_system/` - Agent configuration
- `hand_off_documents/` - Concise handoff notes
- `outputs/` - Stage artifacts (referenced by handoffs)
