## Technical Decisions

### Layer 1: Tool Portability (AGNOSTIC)
- **Decision**: 0AGNOSTIC.md (source) + .0agnostic/ (resources) + agnostic-sync.sh (transformation)
- **Rationale**: Single source of truth, works across tools, lean context
- **Impact**: Tool independence, no lock-in

### Layer 2: Discovery (Traversal)
- **Decision**: 0INDEX.md at branching points + /find skill
- **Rationale**: 20-30 indices cover 5,930 nodes, minimal maintenance
- **Impact**: Automatic discovery in 3-5 steps, no manual navigation

### Layer 3: Safety (Sync)
- **Decision**: File locking + atomic writes + hash-based change detection + last-write-wins resolution
- **Rationale**: Prevents 80% of conflicts, detects 20%, resolves deterministically
- **Impact**: Safe parallel execution without complex CRDT

### Layer 4: Continuity (Episodic)
- **Decision**: Timestamped session files + divergence/conflict logs + quarterly compaction
- **Rationale**: Full audit trail, searchable history, bounded storage
- **Impact**: No agent amnesia, complete session continuity

## Notes & Observations

### Design Philosophy Emerged

The design embodies these principles:

1. **Simplicity Over Cleverness**
   - File locking simpler than full CRDT
   - Markdown indices simpler than graph databases
   - Timestamps simpler than version vectors

2. **Prevention Over Remediation**
   - Prevent conflicts (locking) before trying to resolve
   - Prevent amnesia (episodic memory) before trying to recover

3. **Tool Agnostic Always**
   - No hard dependency on Claude, AutoGen, or Gemini
   - Works with any AI tool via agnostic-sync.sh

4. **Audit Trail Everything**
   - Every session logged
   - Every conflict recorded
   - Full git history maintained
