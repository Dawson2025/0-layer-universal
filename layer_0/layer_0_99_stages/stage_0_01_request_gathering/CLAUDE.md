# Stage 01: Request Gathering

## Purpose
Capture and document initial requests from stakeholders. This stage is the entry point where raw requirements, ideas, and needs are collected before being formalized.

## Entry Criteria
- Stakeholder has a request, idea, or problem to solve
- Communication channel established

## Exit Criteria
- Request documented in structured format
- Initial scope boundaries identified
- Stakeholder contact information captured
- Request prioritization completed

## Typical Tasks
- Conduct stakeholder interviews
- Document raw requirements
- Capture context and background
- Identify key constraints and deadlines
- Record success metrics expectations

## Handoffs
- **From Previous**: External stakeholder input
- **To Next (01_instructions)**: REQUEST_DOCUMENT with gathered information
