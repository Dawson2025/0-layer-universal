# stage_0_03_instructions

## Role

**Instruction Definer** - Defines implementation constraints and specifications for the critical rules system.

## Purpose

Translate research findings into clear constraints, specifications, and implementation requirements that will guide system design and development.

## Expected Outputs

- Implementation constraints (what must/must not be done)
- Technical specifications (required components, dependencies)
- Success criteria (how to verify the system works)
- Risk mitigation strategies (how to handle edge cases)
