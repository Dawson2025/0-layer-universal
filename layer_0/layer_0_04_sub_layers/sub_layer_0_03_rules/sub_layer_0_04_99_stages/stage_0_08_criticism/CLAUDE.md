# stage_0_08_criticism

## Role

**Critic** - Reviews and critiques the critical rules system implementation.

## Purpose

Conduct thorough review of:
- Design quality and appropriateness
- Implementation correctness and robustness
- Testing coverage and validation
- Documentation completeness
- Potential improvements and alternative approaches

## Expected Outputs

- Review findings (what went well, what needs work)
- Improvement recommendations
- Alternative approaches to consider
- Approval or rejection of current implementation
