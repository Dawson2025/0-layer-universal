# Canonical Entity Structure

This is the **single source of truth** for entity directory structure in the layer-stage system. All entity creation tools, skills, guides, and constraints reference this file.

## Full Entity Structure

Every entity (project, feature, component, research project) follows this canonical structure. `N` is the entity's own layer number; `N+1` is the child layer.

```
<entity>/
├── 0AGNOSTIC.md                    # Source of truth (tool-agnostic context)
├── 0INDEX.md                       # Navigation index
├── CLAUDE.md                       # Auto-generated by agnostic-sync.sh
├── AGENTS.md                       # Auto-generated by agnostic-sync.sh
├── GEMINI.md                       # Auto-generated by agnostic-sync.sh
├── OPENAI.md                       # Auto-generated by agnostic-sync.sh
├── .cursorrules                    # Auto-generated by agnostic-sync.sh
│
├── .0agnostic/                     # On-demand AI resources
│   ├── agents/
│   ├── episodic/
│   │   ├── sessions/
│   │   └── changes/
│   ├── hooks/scripts/
│   ├── knowledge/
│   ├── rules/
│   └── skills/
│
├── .1merge/                        # Tool-specific overrides (3-tier)
│   ├── .1claude_merge/
│   │   ├── 0_synced/
│   │   ├── 1_overrides/
│   │   └── 2_additions/
│   ├── .1cursor_merge/{0_synced,1_overrides,2_additions}/
│   ├── .1gemini_merge/{0_synced,1_overrides,2_additions}/
│   ├── .1aider_merge/{0_synced,1_overrides,2_additions}/
│   ├── .1codex_merge/{0_synced,1_overrides,2_additions}/
│   └── .1copilot_merge/{0_synced,1_overrides,2_additions}/

│   └── rules/

│   └── rules/

│   └── instructions/
│
├── layer_N_group/                  # Entity internals (same layer number)
│   ├── layer_N_00_layer_registry/
│   │   └── proposals/
```

## mkdir Command Template

Replace `N` with the entity's layer number (0, 1, 2, -1, etc.) and `N1` with N+1:

```bash
mkdir -p <entity>/{.0agnostic/{agents,episodic/{sessions,changes},hooks/scripts,knowledge,rules,skills},.1merge/{.1claude_merge/{0_synced,1_overrides,2_additions},.1cursor_merge/{0_synced,1_overrides,2_additions},.1gemini_merge/{0_synced,1_overrides,2_additions},.1aider_merge/{0_synced,1_overrides,2_additions},.1codex_merge/{0_synced,1_overrides,2_additions},.1copilot_merge/{0_synced,1_overrides,2_additions}},.claude/rules,.cursor/rules,.github/instructions,layer_N_group/{layer_N_00_layer_registry/proposals,layer_N_01_ai_manager_system,layer_N_02_manager_handoff_documents/{incoming/{from_above,from_below},outgoing/{to_above,to_below}},layer_N_03_sub_layers/{sub_layer_N_00_sub_layer_registry,sub_layer_N_01_prompts,sub_layer_N_02_knowledge_system/{overview,things_learned},sub_layer_N_03_principles,sub_layer_N_04_rules,sub_layer_N_05+_setup_dependant},layer_N_99_stages/stage_N_02_research/outputs/by_topic},layer_N1_group/{layer_N1_00_layer_registry/proposals},synthesis}
```

## Required Files

After creating the directory structure, create these files:

| File | Purpose | How |
|------|---------|-----|
| `0AGNOSTIC.md` | Identity, triggers, pointers | Write manually (see INSTANTIATION_GUIDE.md for template) |
| `0INDEX.md` | Navigation index | Write manually |
| `CLAUDE.md` etc. | Tool-specific context | Run `agnostic-sync.sh` to generate |

## Stage Completeness Rule

If the entity uses stages, create ALL 11:

```bash
for i in 01_request_gathering 02_research 03_instructions 04_planning 05_design 06_development 07_testing 08_criticism 09_fixing 10_current_product 11_archives; do
  mkdir -p "layer_N_group/layer_N_99_stages/stage_N_$i/outputs/{by_topic,episodic/{sessions,changes}}"
done
```

**Empty stages are valid. Missing stages are NOT.**

## Key Conventions

- Stages go inside `layer_N_group/layer_N_99_stages/`, NOT at entity root
- Sub-layers go inside `layer_N_group/layer_N_03_sub_layers/`, NOT at entity root
- Children go in `layer_N+1_group/`, with the child's own layer number = N+1
- `N` always matches the entity's own layer: project=1, feature=2, research feature=0, etc.
- Knowledge system always has `overview/` and `things_learned/` subdirectories
