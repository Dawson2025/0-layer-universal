# Canonical Entity Structure

This is the **single source of truth** for entity directory structure in the layer-stage system. All entity creation tools, skills, guides, and constraints reference this file.

## Full Entity Structure

Every entity (project, feature, component, research project) follows this canonical structure. `N` is the entity's own layer number; `N+1` is the child layer.

```
<entity>/
├── 0AGNOSTIC.md                           # Source of truth (tool-agnostic context)
├── 0INDEX.md                              # Navigation index
├── README.md                              # Human-readable overview
├── CLAUDE.md                              # Auto-generated by agnostic-sync.sh
├── AGENTS.md                              # Auto-generated by agnostic-sync.sh
├── GEMINI.md                              # Auto-generated by agnostic-sync.sh
├── OPENAI.md                              # Auto-generated by agnostic-sync.sh
├── .cursorrules                           # Auto-generated by agnostic-sync.sh
│
├── layer_N_orchestrator.gab.jsonld        # Entity-level orchestrator (GAB agent definition)
├── layer_N_orchestrator.integration.md    # Auto-generated by jsonld-to-md.sh (DO NOT EDIT)
│
├── .0agnostic/                            # On-demand AI resources (SOURCE OF TRUTH for episodic_memory)
│   ├── agents/
│   ├── episodic_memory/                   # Source of truth — syncs to tool dirs
│   │   ├── sessions/
```

## Stage Structure

Each stage has its own config directories, agent definition, and integration summary:

```
stage_N_XX_name/
├── 0AGNOSTIC.md                           # Stage-level context
├── CLAUDE.md                              # Auto-generated by agnostic-sync.sh
├── AGENTS.md                              # Auto-generated by agnostic-sync.sh
├── GEMINI.md                              # Auto-generated by agnostic-sync.sh
├── OPENAI.md                              # Auto-generated by agnostic-sync.sh
├── stage_N_XX_name_agent.jsonld           # Stage agent definition
├── stage_N_XX_name_agent.integration.md   # Auto-generated by jsonld-to-md.sh (DO NOT EDIT)
├── outputs/                               # Stage outputs
├── .0agnostic/                            # Same structure as entity root .0agnostic/
│   ├── agents/
│   ├── episodic_memory/{sessions,changes}/
│   ├── hooks/scripts/
│   ├── knowledge/
│   ├── rules/
│   └── skills/
├── .1merge/                               # Same structure as entity root .1merge/
│   └── {6 tools × 3 tiers each}
├── .claude/{rules,episodic_memory/{sessions,changes}}/
├── .cursor/{rules,episodic_memory/{sessions,changes}}/
├── .gemini/episodic_memory/{sessions,changes}/
├── .codex/episodic_memory/{sessions,changes}/
├── .github/instructions/
└── synthesis/
```

## Internal Directories Also Need 0AGNOSTIC.md

Every directory inside `layer_N_group/` should have its own `0AGNOSTIC.md` + auto-generated tool files:

- `layer_N_group/layer_N_00_layer_registry/0AGNOSTIC.md`
- `layer_N_group/layer_N_01_ai_manager_system/0AGNOSTIC.md`
- `layer_N_group/layer_N_02_manager_handoff_documents/0AGNOSTIC.md`
- `layer_N_group/layer_N_03_sub_layers/0AGNOSTIC.md` (and each sub_layer_N_XX/)
- `layer_N_group/layer_N_99_stages/0AGNOSTIC.md` (and each stage_N_XX/)

Run `agnostic-sync.sh` on each `0AGNOSTIC.md` to generate CLAUDE.md, AGENTS.md, GEMINI.md, OPENAI.md.

## Agent Files: Every .jsonld MUST Have a .integration.md

**Rule**: Every `.gab.jsonld` or `.agent.jsonld` file MUST have a matching `.integration.md` file with the same base name.

| Agent File | Integration File | Generated By |
|---|---|---|
| `layer_N_orchestrator.gab.jsonld` | `layer_N_orchestrator.integration.md` | `tools/jsonld-to-md.sh` |
| `layer_N_99_stages_orchestrator.gab.jsonld` | `layer_N_99_stages_orchestrator.integration.md` | `tools/jsonld-to-md.sh` |
| `stage_N_XX_name_agent.jsonld` | `stage_N_XX_name_agent.integration.md` | `tools/jsonld-to-md.sh` |

**Generate**: `bash tools/jsonld-to-md.sh <path-to-jsonld>` then rename `*.jsonld.integration.md` → `*.integration.md`

## Naming Conventions

| Convention | Rule | Example |
|---|---|---|
| **Internal layer dir** | MUST use `_group` suffix | `layer_7_group/` (NOT `layer_7/`) |
| **Children dir** | MUST use `_group` suffix | `layer_8_group/` (NOT `layer_8/`) |
| **Episodic memory** | MUST be named `episodic_memory` | `.0agnostic/episodic_memory/` (NOT `episodic/`) |
| **Agent files** | Use dot-separation | `name.gab.jsonld` (NOT `name_gab.jsonld`) |
| **Entity naming** | `layer_{N}_{type}_{name}` | `layer_2_feature_assignments` |
| **Setup hierarchy grouping** | Use `_group` suffix (NOT `_content`) | `sub_layer_0_05_group/` |
| **Setup hierarchy items** | Use `sub_layer_` prefix | `sub_layer_0_05_linux_ubuntu` (NOT `linux_ubuntu`) |

## mkdir Command Template

Replace `N` with the entity's layer number and `N1` with N+1:

```bash

mkdir -p <entity>/{.0agnostic/{agents,episodic_memory/{sessions,changes},hooks/scripts,knowledge,rules,skills},.1merge/{.1claude_merge/{0_synced,1_overrides,2_additions},.1cursor_merge/{0_synced,1_overrides,2_additions},.1gemini_merge/{0_synced,1_overrides,2_additions},.1aider_merge/{0_synced,1_overrides,2_additions},.1codex_merge/{0_synced,1_overrides,2_additions},.1copilot_merge/{0_synced,1_overrides,2_additions}},.claude/{rules,episodic_memory/{sessions,changes}},.cursor/{rules,episodic_memory/{sessions,changes}},.gemini/episodic_memory/{sessions,changes},.codex/episodic_memory/{sessions,changes},.github/instructions}

# Internal layer_N_group structure
mkdir -p <entity>/layer_N_group/{layer_N_00_layer_registry/proposals,layer_N_01_ai_manager_system,layer_N_02_manager_handoff_documents/{incoming/{from_above,from_below},outgoing/{to_above,to_below}},layer_N_03_sub_layers/{sub_layer_N_00_sub_layer_registry,sub_layer_N_01_knowledge_system/{overview,things_learned,principles},sub_layer_N_02_rules/{static,dynamic},sub_layer_N_03_protocols,sub_layer_N_04+_setup_dependant},layer_N_99_stages}
```

## Stage Creation Template

Create ALL 12 stages (00-11). **Empty stages are valid. Missing stages are NOT.**

```bash
# Create stage 00 (registry)
mkdir -p "layer_N_group/layer_N_99_stages/stage_N_00_stage_registry/{outputs,synthesis}"

# Create stages 01-11 (each gets outputs/ and synthesis/)
for i in 01_request_gathering 02_research 03_instructions 04_planning 05_design 06_development 07_testing 08_criticism 09_fixing 10_current_product 11_archives; do
  mkdir -p "layer_N_group/layer_N_99_stages/stage_N_$i/{outputs,synthesis}"
done

# Add config directories to EACH stage
for stage_dir in layer_N_group/layer_N_99_stages/stage_N_*/; do
  mkdir -p "$stage_dir"/{.0agnostic/{agents,episodic_memory/{sessions,changes},hooks/scripts,knowledge,rules,skills},.1merge/{.1claude_merge/{0_synced,1_overrides,2_additions},.1cursor_merge/{0_synced,1_overrides,2_additions},.1gemini_merge/{0_synced,1_overrides,2_additions},.1aider_merge/{0_synced,1_overrides,2_additions},.1codex_merge/{0_synced,1_overrides,2_additions},.1copilot_merge/{0_synced,1_overrides,2_additions}},.claude/{rules,episodic_memory/{sessions,changes}},.cursor/{rules,episodic_memory/{sessions,changes}},.gemini/episodic_memory/{sessions,changes},.codex/episodic_memory/{sessions,changes},.github/instructions,synthesis}
done
```

## Required Files

After creating the directory structure, create these files:

| File | Location | Purpose | How |
|------|----------|---------|-----|
| `0AGNOSTIC.md` | Entity root | Identity, triggers, pointers | Write manually (see INSTANTIATION_GUIDE.md) |
| `0INDEX.md` | Entity root | Navigation index | Write manually |
| `README.md` | Entity root | Human-readable overview | Write manually |
| `CLAUDE.md` etc. | Entity root | Tool-specific context | Run `agnostic-sync.sh` |
| `0AGNOSTIC.md` | Every internal dir | Directory-level context | Write manually, run agnostic-sync.sh |
| `layer_N_orchestrator.gab.jsonld` | Entity root | Entity orchestrator | Copy from sibling, adapt |
| `layer_N_orchestrator.integration.md` | Entity root | Readable summary | Run `jsonld-to-md.sh` |
| `layer_N_99_stages_orchestrator.gab.jsonld` | `layer_N_99_stages/` | Stages orchestrator | Create or copy from sibling |
| `layer_N_99_stages_orchestrator.integration.md` | `layer_N_99_stages/` | Readable summary | Run `jsonld-to-md.sh` |
| `stage_N_XX_name_agent.jsonld` | Each stage dir | Stage agent stub | Create per stage |
| `stage_N_XX_name_agent.integration.md` | Each stage dir | Readable summary | Run `jsonld-to-md.sh` |
| `status_N.json` | `layer_N_99_stages/` | Stage workflow tracker | Create with stage status |

## Key Conventions

- Internal content uses `layer_N_group/` with the **_group suffix** — NOT bare `layer_N/`
- Stages go inside `layer_N_group/layer_N_99_stages/`, NOT at entity root
- Sub-layers go inside `layer_N_group/layer_N_03_sub_layers/`, NOT at entity root
- Children go in `layer_N+1_group/`, with the child's own layer number = N+1
- `N` always matches the entity's own layer: project=1, feature=2, research feature=0, etc.
- Knowledge system always has `overview/`, `things_learned/`, and `principles/` subdirectories
- Rules always has `static/` (always-on constraints) and `dynamic/` (trigger-based with protocol pointers) subdirectories
- Setup-dependant hierarchy nodes include `knowledge_system/`, `rules/`, `protocols/` triplet at each specific node and `_shared/` dir
- Episodic memory is always named `episodic_memory/` (NOT `episodic/`)
- `.0agnostic/episodic_memory/` is the source of truth — tool dirs receive synced copies
- `outputs/` and `synthesis/` live inside each **stage**, NOT at entity root
- Every tool dir (`.claude/`, `.cursor/`, `.gemini/`, `.codex/`) gets its own `episodic_memory/`
