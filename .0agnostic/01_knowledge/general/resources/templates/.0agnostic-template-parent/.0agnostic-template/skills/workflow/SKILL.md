# Workflow Skill

<!-- section_id: "8d4d9fc1-0890-468a-8b59-1d7e28f667ea" -->
## Purpose

This skill provides guidance for how to work effectively at this location in the layer-stage system.

<!-- section_id: "dcca61cf-b6fc-4440-bf01-b041efd63927" -->
## When to Use

- Starting work at this location
- Unsure of the process for this stage
- Need to understand outputs expected

<!-- section_id: "31b3e4e4-ac02-4993-b031-73b0467e8000" -->
## Workflow Steps

<!-- section_id: "822d035d-1156-42d2-b2ac-fa5257c5f906" -->
### 1. Check Context
- Read `AGNOSTIC.md` (or `CLAUDE.md`)
- Check `hand_off_documents/incoming/` for tasks
- Review recent `outputs/` for context

<!-- section_id: "51351492-b18b-49a3-a3b2-42d8b6761ba7" -->
### 2. Understand Scope
- What stage is this? (01-11)
- What layer is this? (L-1, L0, L1)
- What can be done here vs. escalated?

<!-- section_id: "5cb2500e-c1d7-4c58-aed7-9aae90146d09" -->
### 3. Do Work
- Follow stage-specific process
- Write outputs to `outputs/`
- Update episodic memory if enabled

<!-- section_id: "e4ca7987-6366-4398-9ecb-cdaca5aed86c" -->
### 4. Complete or Escalate
- If done: Write to `hand_off_documents/outgoing/`
- If blocked: Escalate to parent with context
