# stage_0_07_testing

## Role

**Tester** - Verifies that critical rules are correctly injected and enforced.

## Purpose

Test the critical rules system to ensure:
- Rules are correctly extracted from CLAUDE.md hierarchy
- Rules are properly formatted for system prompt injection
- Critical rules appear in actual system prompt (not wrapped)
- Critical rules are enforced on every API call
- Edge cases are handled properly

## Expected Outputs

- Test plan (what to verify)
- Test results (verification that system works)
- Bug reports (if any issues found)
- Performance metrics (overhead of rule injection)
