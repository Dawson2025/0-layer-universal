# stage_0_04_design

## Role

**Architect** - Designs the architecture for the dynamic critical rules injection system.

## Purpose

Create detailed architecture and design specifications for the critical rules system, including:
- System components and their interactions
- Data structures and formats
- API/interface specifications
- Integration points with Claude Code

## Expected Outputs

- Architecture diagram (system components)
- Component specifications (what each part does)
- Data flow diagram (how rules flow through system)
- Integration specifications (how it connects to Claude Code)
- Design rationale (why choices were made)
