# Browser MCP Routing Table (CLI × OS × Task)

This file is about **recommendations** (which browser MCP to use). For what is **currently automated/configured**, see `MCP_SERVER_MATRIX.md`.

<!-- section_id: "e80e9f1e-bab5-4afe-b13c-8f1303b055e4" -->
## Primary rule
Default to **Playwright MCP** for browser automation + page extraction. Add other servers only when you need specific capabilities (DevTools debugging, hosted/proxied browsers, etc.).

<!-- section_id: "492a4013-b23a-4f74-b539-33ed5bec791b" -->
## Capabilities map (quick)
- **Playwright MCP (`@playwright/mcp`)**: best general browser control + extraction.
- **Chrome DevTools MCP (`chrome-devtools-mcp`)**: best for DevTools-level debugging/inspection (console/network/perf) and attaching to an existing Chrome.
- **Browser MCP (`@agent-infra/mcp-server-browser`)**: Playwright-based; use when its tool schema works better for a given client.
- **Hosted browser MCPs (Browserbase, etc.)**: use when local headed display is unreliable or you need proxies/geo/isolation.

<!-- section_id: "567f2c9f-f5bd-4268-9e5e-d05ec0161e8b" -->
## Routing table (recommended defaults)

<!-- section_id: "eca35e00-c5b9-4a91-b623-57af6c6bc668" -->
### Codex CLI
| OS/Environment | Default (browser tasks) | Fallback | Notes |
|---|---|---|---|
| Linux desktop | Playwright MCP (headed) | Chrome DevTools MCP | Prefer headed for interactive navigation; use DevTools MCP for debugging |
| WSL/No GUI | Playwright MCP (headless) | Hosted browser MCP | Headed requires working DISPLAY/Wayland bridge; otherwise go headless/hosted |
| Remote/SSH | Playwright MCP (headless) | Hosted browser MCP | Use hosted for stable long sessions |

<!-- section_id: "d3ac278b-1f4e-479c-ae70-8a64d9c75898" -->
### Gemini CLI
| OS/Environment | Default (browser tasks) | Fallback | Notes |
|---|---|---|---|
| Linux desktop | Playwright MCP (headed) | Browser MCP | Gemini often works well with Playwright; Browser MCP is optional schema alternative |
| WSL/No GUI | Playwright MCP (headless) | Hosted browser MCP | Prefer headless or hosted to avoid DISPLAY issues |

<!-- section_id: "2744b19c-646e-47e5-9291-f2c64aa3e22f" -->
### Claude Code (CLI)
| OS/Environment | Default (browser tasks) | Fallback | Notes |
|---|---|---|---|
| Linux desktop | Playwright MCP (headed) | Chrome DevTools MCP | Add DevTools MCP for deep inspection/tracing |
| WSL/No GUI | Playwright MCP (headless) | Hosted browser MCP | Same constraints as others |

## Task-based overrides

<!-- section_id: "e2bae438-646f-4289-b0d7-87e35e5ee5e9" -->
### “I need to click around + pull content from pages”
- Use: **Playwright MCP**
- Notes: prefer headed only if you need a human to take over navigation mid-run.

<!-- section_id: "6406c92e-75f9-4685-92bc-a2aa9f733743" -->
### “I need console logs / network requests / performance debugging”
- Use: **Chrome DevTools MCP**
- Notes: requires Chrome with remote debugging enabled (or a DevTools-capable setup).

<!-- section_id: "ef81b363-111c-4e6b-9fca-ea572fae27e7" -->
### “Local headed browser isn’t visible / GUI is flaky”
- Use: **Playwright MCP headless**
- If you require “headed” visibility reliably: switch to a **hosted browser MCP**.
  - On WSLg, if headed crashes immediately, launch Playwright MCP with `--config` and add Chromium args `--ozone-platform=wayland` and `--enable-features=UseOzonePlatform`.

<!-- section_id: "fd53a92c-c25b-41e0-aa14-e757d7a8c255" -->
### “I need proxies / geo / hardened isolation”
- Use: **hosted browser MCP** (Browserbase-class)
- Notes: keep Playwright MCP as fallback when hosted isn’t available.

## Implementation hooks (where this gets enforced)
- **Codex CLI**: `~/.codex/config.toml` generated by `automation/scripts/codex_mcp_sync.py`
  - Headed vs headless should be selected by flags (`--headless`) and/or OS detection.
- **Gemini CLI**: `~/.gemini/settings.json` `mcpServers` section (single canonical file pattern).
- **Claude Code**: its MCP config (client-specific) should mirror the same routing defaults.

<!-- section_id: "0d10084b-735f-4d31-a3fb-e85ab6973ba3" -->
## Operational checklist (every browser session)
1. Confirm environment: Linux desktop vs WSL/no GUI.
2. Pick MCP server from the table above.
3. Ensure required env vars exist (DISPLAY/PLAYWRIGHT_BROWSERS_PATH/API keys).
4. Restart the CLI after changing MCP config.
