# AI Development Frameworks
*Comprehensive Documentation for Spec-Driven and Agentic Development Workflows*

<!-- section_id: "7a0d6af8-d25d-4e7a-b1f2-aba4d0d79f0e" -->
## Overview

This directory contains documentation for advanced frameworks designed to structure and optimize AI-driven development workflows. These frameworks transform ad-hoc prompting into systematic, verifiable development processes.

<!-- section_id: "32ec801a-9e7e-46b3-999c-33da9f3cc472" -->
## Framework Categories

<!-- section_id: "1b32d7b2-0171-4dfa-bcb0-1b80757329ea" -->
### 1. Spec-Driven Development
**Primary Framework**: GitHub Spec Kit
- Structured 4-phase validation process
- Version-controlled specifications
- Works with GitHub Copilot, Claude Code, Gemini CLI

<!-- section_id: "d21e4a9c-5eec-4da9-802b-f49bf41487ab" -->
### 2. Agentic Team Development
**Primary Framework**: BMAD Method
- Specialized AI agents as development team
- Git-based artifact versioning
- Context-engineered workflows

<!-- section_id: "c2475e74-e117-4541-a636-374e34985b44" -->
### 3. AI Coding Assistants
**Primary Tools**: Cursor, Windsurf, Copilot, Aider, Claude Code, Qwen3
- IDE-integrated AI assistants
- Terminal-based pair programmers
- Design-to-code tools
- Privacy-focused local tools

## Quick Reference

<!-- section_id: "38ca65d7-f1c4-4d7b-abbc-fbdd674fe3a5" -->
### When to Use Spec Kit
- ✅ You want structured, predictable development workflows
- ✅ You need version-controlled specifications
- ✅ You prefer 4-phase gated development with validation checkpoints
- ✅ You work with GitHub Copilot, Claude Code, or Gemini CLI

<!-- section_id: "4c4e18ea-07da-4fec-ad32-4e2c15217273" -->
### When to Use BMAD Method
- ✅ You want agentic team collaboration
- ✅ You need specialized roles (Analyst, Product Manager, Architect, etc.)
- ✅ You prefer context-engineered development with detailed stories
- ✅ You want audit trails and traceability

<!-- section_id: "641cc544-304e-46f7-90eb-244fcd9c83eb" -->
### When to Use Both
- ✅ You want comprehensive development coverage
- ✅ You need both structured workflows and team collaboration
- ✅ You're building complex, multi-faceted projects
- ✅ You want maximum context preservation

<!-- section_id: "0ebfd41b-f4bf-4862-bb82-e168051ad05a" -->
## Framework Features Comparison

| Feature | Spec Kit | BMAD Method |
|---------|----------|-------------|
| **Approach** | Spec-driven workflows | Agentic team collaboration |
| **Validation** | 4-phase gated process | Pull request reviews (human + AI) |
| **Specifications** | Version-controlled alongside code | Git-based artifact versioning |
| **AI Agents** | Single AI with structured prompts | Multiple specialized agents |
| **Traceability** | Structured checkpoints | Comprehensive audit trails |
| **Context Management** | Clear, structured guidance | Context-engineered development stories |

## Best Practices

<!-- section_id: "8d942407-76d2-4847-b6ea-2904d08d2413" -->
### Framework Selection
1. **Assess Your Needs**: What problems are you trying to solve?
2. **Consider Team Size**: Small teams may prefer Spec Kit, larger teams may benefit from BMAD
3. **Evaluate Complexity**: Simple projects = Spec Kit, Complex projects = Both
4. **Test Both**: Start with pilot projects to determine fit

<!-- section_id: "d9e4eb97-3106-4add-a199-fddf02532f17" -->
### Implementation
1. **Start Small**: Begin with a proof-of-concept project
2. **Follow Documentation**: Each framework has comprehensive guides
3. **Iterate**: Learn from your experience and refine your workflow
4. **Share Knowledge**: Document what works and what doesn't

<!-- section_id: "7e9df24d-0b89-4db4-b35f-dcad191f27e7" -->
### Maintenance
1. **Keep Updated**: Follow framework updates and improvements
2. **Version Control**: Use Git for all artifacts and specifications
3. **Document Learnings**: Maintain team knowledge base
4. **Review Regularly**: Assess effectiveness and adjust
