## File Documentation and Organization – Workflows

This folder is for **concrete workflows** that apply the File Documentation and Organization Protocol, for example:

- “Document a large research chat into `chat_history/things_learned/overview`.”
- “Refactor legacy docs into layered specs and overviews.”

Each workflow instance here can follow the standard pattern:

- `0.02_sub_layers/` – helpers, templates, or per-topic variations.
- `0.99_stages/` – request_gathering → instructions → planning → design → implementation → testing → criticism → fixing → archiving, each with `ai_agent_system/` and `hand_off_documents/`.
