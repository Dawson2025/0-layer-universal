## Small Batch Verification – Workflows

This folder is for **concrete workflows** that apply the Small Batch Verification Protocol in specific contexts (data extraction, refactoring, testing, etc.).

Each workflow instance you create here (e.g., `workflow_1_data_extraction/`, `workflow_2_refactor_auth/`) can follow the standard pattern:

- `0.02_sub_layers/` – any internal sub-layers or helper materials.
- `0.99_stages/` – request_gathering → instructions → planning → design → implementation → testing → criticism → fixing → archiving, each with `ai_agent_system/` and `hand_off_documents/`.

Use this area when you want to turn the abstract Small Batch Verification Protocol into repeatable, scripted workflows.
