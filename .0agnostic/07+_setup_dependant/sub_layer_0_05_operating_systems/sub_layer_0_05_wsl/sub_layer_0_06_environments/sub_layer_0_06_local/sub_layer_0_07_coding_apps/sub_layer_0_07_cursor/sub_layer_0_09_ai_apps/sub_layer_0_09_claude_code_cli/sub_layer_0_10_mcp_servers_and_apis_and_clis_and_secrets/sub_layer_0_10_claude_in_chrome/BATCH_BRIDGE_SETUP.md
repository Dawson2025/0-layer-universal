# Claude in Chrome Extension → WSL Claude Code Bridge Setup

<!-- section_id: "d8288e13-a468-4af8-8de2-57d6f508727d" -->
## Overview

This guide sets up a Native Messaging bridge to enable the **Claude in Chrome** extension (running on Windows Chrome) to communicate with **Claude Code CLI** running in WSL (Windows Subsystem for Linux).

<!-- section_id: "6e9ea555-5646-401c-b2ba-e6bef7343a19" -->
### Architecture

```
Windows Chrome Extension
        ↓
Windows Batch Script (claude-chrome-host.bat)
        ↓
WSL Bash Wrapper (claude-chrome-host.sh)
        ↓
Claude Code Native Host (~/.claude/chrome/chrome-native-host)
        ↓
Claude Code CLI (with --chrome-native-host flag)
```

<!-- section_id: "f2b7a4f7-b2e7-484d-88bf-c6bed769d882" -->
### How It Works

1. Chrome extension sends Native Messaging protocol messages (32-bit length prefix + JSON)
2. Windows batch script receives stdin/stdout and pipes to WSL
3. WSL wrapper calls the actual Claude Code native host
4. Claude Code processes the request and returns the response
5. Response flows back through the chain to Chrome

## Prerequisites

- Windows 10/11 with WSL2 installed
- Claude Code CLI installed in WSL (`npm install -g @anthropic-ai/claude-code`)
- Chrome or Edge browser on Windows
- Claude in Chrome extension installed from Chrome Web Store

## Installation Steps

<!-- section_id: "a470e668-c6a6-4941-97aa-76729d7d3b90" -->
### Step 1: Verify WSL Setup

First, verify that Claude Code is installed and the native host exists:

```bash
# In WSL terminal
which claude
# Should output: /home/dawson/.nvm/versions/node/v22.20.0/bin/claude

ls -l ~/.claude/chrome/chrome-native-host
# Should exist and be executable
```

If the native host doesn't exist, run Claude Code once with the `--chrome` flag to generate it:

```bash
claude --chrome
# Press Ctrl+C to exit after it starts
```

<!-- section_id: "7077d5df-7046-4cb8-bc0f-6f761caa4826" -->
### Step 2: Create WSL Wrapper Script

The wrapper script at `~/bin/claude-chrome-host.sh` has already been created. Verify it exists:

```bash
cat ~/bin/claude-chrome-host.sh
```

If it doesn't exist, create it:

```bash
mkdir -p ~/bin
cat > ~/bin/claude-chrome-host.sh << 'EOFSCRIPT'
#!/bin/bash
# WSL wrapper for Claude Code Native Messaging Host
# This script is called by the Windows batch script to bridge Chrome -> WSL
# Generated for Claude in Chrome extension integration

# Enable error logging (optional - comment out for production)
# exec 2>> /home/dawson/claude-chrome-host.log
# echo "[$(date)] Starting claude-chrome-host" >&2

# Get the absolute path to the actual native host script
NATIVE_HOST="/home/dawson/.claude/chrome/chrome-native-host"

# Check if the native host exists
if [ ! -f "$NATIVE_HOST" ]; then
    echo "[$(date)] ERROR: Native host not found at $NATIVE_HOST" >&2
    exit 1
fi

# Execute the native host
# This script handles the Native Messaging protocol (stdin/stdout with 32-bit length prefix + JSON)
exec "$NATIVE_HOST"
EOFSCRIPT

chmod +x ~/bin/claude-chrome-host.sh
```

<!-- section_id: "78674d9b-6bc5-4858-a5f0-3d9d424900f8" -->
### Step 3: Create Windows Batch Script

Create the Windows batch script that Chrome will call. This needs to be created on the **Windows filesystem**.

**Location:** `C:\Users\<YourUsername>\bin\claude-chrome-host.bat`

**Content:**

```batch
@echo off
REM Claude in Chrome Native Messaging Host Bridge
REM Bridges Windows Chrome -> WSL Claude Code CLI
REM Generated by Claude Code - do not edit manually

REM Call WSL and execute the wrapper script
REM stdin/stdout are piped directly to maintain Native Messaging protocol
wsl.exe -e /home/dawson/bin/claude-chrome-host.sh
```

**To create this file:**

1. Open PowerShell as Administrator
2. Run:
   ```powershell
   New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\bin"
   
   @"
   @echo off
   REM Claude in Chrome Native Messaging Host Bridge
   REM Bridges Windows Chrome -> WSL Claude Code CLI
   REM Generated by Claude Code - do not edit manually

   REM Call WSL and execute the wrapper script
   REM stdin/stdout are piped directly to maintain Native Messaging protocol
   wsl.exe -e /home/dawson/bin/claude-chrome-host.sh
   "@ | Out-File -FilePath "$env:USERPROFILE\bin\claude-chrome-host.bat" -Encoding ASCII
   ```

<!-- section_id: "b2ceee94-9f12-43ba-9988-a8fc4015eb07" -->
### Step 4: Find Chrome Extension ID

1. Open Chrome and go to `chrome://extensions/`
2. Enable "Developer mode" (toggle in top-right)
3. Find "Claude in Chrome" extension
4. Copy the extension ID (e.g., `abcdefghijklmnopqrstuvwxyz123456`)

**Alternative:** Check the extension's page source or manifest. The ID is a 32-character string.

<!-- section_id: "04e1e1e0-81fd-482c-8cc0-e47d3af445c5" -->
### Step 5: Create Native Messaging Manifest

Create the Native Messaging manifest JSON file that tells Chrome how to communicate with the host.

**Location (Chrome):** 
```
%USERPROFILE%\AppData\Local\Google\Chrome\User Data\NativeMessagingHosts\com.anthropic.claude.chrome.json
```

**Location (Edge):**
```
%USERPROFILE%\AppData\Local\Microsoft\Edge\User Data\NativeMessagingHosts\com.anthropic.claude.chrome.json
```

**Manifest Content:**

```json
{
  "name": "com.anthropic.claude.chrome",
  "description": "Claude in Chrome native messaging host",
  "path": "C:\\Users\\<YourUsername>\\bin\\claude-chrome-host.bat",
  "type": "stdio",
  "allowed_origins": [
    "chrome-extension://<EXTENSION_ID>/"
  ]
}
```

**To create this file (PowerShell):**

```powershell
# Set your extension ID here
$extensionId = "YOUR_EXTENSION_ID_HERE"  # Replace with actual ID

# For Chrome
$chromeDir = "$env:LOCALAPPDATA\Google\Chrome\User Data\NativeMessagingHosts"
New-Item -ItemType Directory -Force -Path $chromeDir

$manifestPath = "$chromeDir\com.anthropic.claude.chrome.json"
$batchPath = "$env:USERPROFILE\bin\claude-chrome-host.bat".Replace('\', '\\')

@"
{
  "name": "com.anthropic.claude.chrome",
  "description": "Claude in Chrome native messaging host",
  "path": "$batchPath",
  "type": "stdio",
  "allowed_origins": [
    "chrome-extension://$extensionId/"
  ]
}
"@ | Out-File -FilePath $manifestPath -Encoding UTF8

Write-Host "Chrome manifest created at: $manifestPath"

# For Edge (optional)
$edgeDir = "$env:LOCALAPPDATA\Microsoft\Edge\User Data\NativeMessagingHosts"
New-Item -ItemType Directory -Force -Path $edgeDir

$edgeManifestPath = "$edgeDir\com.anthropic.claude.chrome.json"

@"
{
  "name": "com.anthropic.claude.chrome",
  "description": "Claude in Chrome native messaging host",
  "path": "$batchPath",
  "type": "stdio",
  "allowed_origins": [
    "chrome-extension://$extensionId/"
  ]
}
"@ | Out-File -FilePath $edgeManifestPath -Encoding UTF8

Write-Host "Edge manifest created at: $edgeManifestPath"
```

### Step 6: Register in Windows Registry (Alternative Method)

You can also register the native host via Windows Registry instead of placing the manifest directly:

**For Chrome:**

```powershell
$manifestPath = "$env:LOCALAPPDATA\Google\Chrome\User Data\NativeMessagingHosts\com.anthropic.claude.chrome.json"

# Create registry key
$regPath = "HKCU:\Software\Google\Chrome\NativeMessagingHosts\com.anthropic.claude.chrome"
New-Item -Path $regPath -Force | Out-Null
New-ItemProperty -Path $regPath -Name "(Default)" -Value $manifestPath -PropertyType String -Force
```

**For Edge:**

```powershell
$manifestPath = "$env:LOCALAPPDATA\Microsoft\Edge\User Data\NativeMessagingHosts\com.anthropic.claude.chrome.json"

# Create registry key
$regPath = "HKCU:\Software\Microsoft\Edge\NativeMessagingHosts\com.anthropic.claude.chrome"

New-ItemProperty -Path $regPath -Name "(Default)" -Value $manifestPath -PropertyType String -Force
```

## Troubleshooting

<!-- section_id: "72f94bbc-ba0e-4968-a0a4-969ff71f9273" -->
### Problem: Chrome says "Specified native messaging host not found"

**Solutions:**

1. Verify the manifest path is correct:
   ```powershell
   Get-Content "$env:LOCALAPPDATA\Google\Chrome\User Data\NativeMessagingHosts\com.anthropic.claude.chrome.json"
   ```

2. Check the batch script path in the manifest matches the actual location:
   ```powershell
   Test-Path "$env:USERPROFILE\bin\claude-chrome-host.bat"
   ```

3. Verify the extension ID in the manifest matches the actual extension ID

4. Restart Chrome completely (close all windows)

<!-- section_id: "1ad97398-de38-451c-a889-836252730ae5" -->
### Problem: "Access denied" or "Cannot open file"

**Solutions:**

1. Check file permissions on the batch script:
   ```powershell
   icacls "$env:USERPROFILE\bin\claude-chrome-host.bat"
   ```

2. Run as Administrator if needed

3. Check Windows Defender or antivirus isn't blocking the script

<!-- section_id: "a5a1592d-9761-4c7c-b7ff-279d52e78c7b" -->
### Problem: Native host starts but doesn't respond

**Solutions:**

1. Enable logging in the WSL wrapper:
   ```bash
   # Edit ~/bin/claude-chrome-host.sh
   # Uncomment these lines:
   exec 2>> /home/dawson/claude-chrome-host.log
   echo "[$(date)] Starting claude-chrome-host" >&2
   ```

2. Check the log:
   ```bash
   tail -f ~/claude-chrome-host.log
   ```

3. Verify Claude Code native host is working:
   ```bash
   ~/.claude/chrome/chrome-native-host
   # Should wait for input (press Ctrl+C)
   ```

4. Test the full chain manually:
   ```bash
   # From Windows PowerShell
   echo '{"type":"ping"}' | wsl.exe -e /home/dawson/bin/claude-chrome-host.sh
   ```

<!-- section_id: "9959adac-ece7-4956-b705-15798d3107ea" -->
### Problem: "wsl.exe not found" or WSL errors

**Solutions:**

1. Verify WSL is installed and working:
   ```powershell
   wsl.exe --version
   wsl.exe --list
   ```

2. Ensure WSL2 is set as default:
   ```powershell
   wsl.exe --set-default-version 2
   ```

3. Make sure your WSL distribution is running:
   ```powershell
   wsl.exe -d Ubuntu  # or your distro name
   ```

<!-- section_id: "b34958e1-3990-40c7-a31c-197e196ef5f9" -->
### Problem: Extension connects but requests fail

**Solutions:**

1. Check Claude Code is properly authenticated in WSL:
   ```bash
   claude --version
   # Should not ask for auth
   ```

2. Verify the native host can communicate:
   ```bash
   # Test manually
   echo '{"action":"ping"}' | ~/.claude/chrome/chrome-native-host
   ```

3. Check for MCP server issues:
   ```bash
   claude --debug mcp
   ```

<!-- section_id: "19592942-b743-4097-9c0c-332712e1fa55" -->
### Problem: Native Messaging protocol errors

**Symptoms:** "Invalid message format" or binary data issues

**Solutions:**

1. Ensure the batch script is NOT adding any extra output (like echo statements)
2. Verify binary stdin/stdout handling:
   - The Native Messaging protocol uses binary format (32-bit little-endian length prefix + JSON)
   - Do NOT use text mode or add any echo/print statements
   - The `wsl.exe -e` command should handle binary properly

3. Test the protocol manually:
   ```python
   # Python script to test Native Messaging protocol
   import struct
   import json
   import sys
   
   def send_message(message):
       encoded = json.dumps(message).encode('utf-8')
       sys.stdout.buffer.write(struct.pack('I', len(encoded)))
       sys.stdout.buffer.write(encoded)
       sys.stdout.buffer.flush()
   
   send_message({"type": "ping"})
   ```

## Understanding Native Messaging Protocol

The Native Messaging protocol used by Chrome extensions is binary-based:

<!-- section_id: "88978b5e-3a0a-40af-a252-34f777a05898" -->
### Message Format

Each message consists of:
1. **Length prefix:** 4 bytes (32-bit unsigned integer, little-endian)
2. **JSON message:** UTF-8 encoded JSON string

<!-- section_id: "8f30a4e0-ad79-4314-88b0-daeaceed4215" -->
### Example

To send `{"type":"ping"}`:

```
Bytes: 0x0E 0x00 0x00 0x00  { " t " y " p " e " : " p " i " n " g " }
        └─── Length (14) ───┘  └──────── JSON (14 bytes) ────────────┘
```

<!-- section_id: "4827d812-7404-4293-b054-6acc5eedb74c" -->
### Important Notes

- Maximum message size: 1 MB (1,048,576 bytes)
- stdin/stdout must be in binary mode
- No extra output should be sent to stdout (no echo, print, etc.)
- stderr can be used for logging

   ```bash
   #!/bin/bash
   # Enable detailed logging

       echo "[$(date)] ERROR: Native host not found at $NATIVE_HOST" >&2

   exec "$NATIVE_HOST"
   ```

## Complete Setup PowerShell Script

Here's a complete PowerShell script to automate the Windows setup:

```powershell
# Claude in Chrome WSL Bridge - Automated Setup Script
# Run this in PowerShell as Administrator

# Step 1: Get extension ID
Write-Host "Please enter your Claude in Chrome extension ID:" -ForegroundColor Cyan
Write-Host "Find it at chrome://extensions/ (enable Developer mode)" -ForegroundColor Yellow
$extensionId = Read-Host "Extension ID"

if ($extensionId.Length -ne 32) {
    Write-Host "WARNING: Extension ID should be 32 characters" -ForegroundColor Red
}

# Step 2: Create batch script
$batchDir = "$env:USERPROFILE\bin"
$batchPath = "$batchDir\claude-chrome-host.bat"

New-Item -ItemType Directory -Force -Path $batchDir | Out-Null

@"
@echo off
REM Claude in Chrome Native Messaging Host Bridge
REM Bridges Windows Chrome -> WSL Claude Code CLI
REM Generated by Claude Code - do not edit manually

REM Call WSL and execute the wrapper script
REM stdin/stdout are piped directly to maintain Native Messaging protocol
wsl.exe -e /home/dawson/bin/claude-chrome-host.sh
"@ | Out-File -FilePath $batchPath -Encoding ASCII

Write-Host "`nBatch script created: $batchPath" -ForegroundColor Green

# Step 3: Create Chrome manifest
$chromeDir = "$env:LOCALAPPDATA\Google\Chrome\User Data\NativeMessagingHosts"
New-Item -ItemType Directory -Force -Path $chromeDir | Out-Null

$manifestPath = "$chromeDir\com.anthropic.claude.chrome.json"
$batchPathEscaped = $batchPath.Replace('\', '\\')

@"
{
  "name": "com.anthropic.claude.chrome",
  "description": "Claude in Chrome native messaging host",
  "path": "$batchPathEscaped",
  "type": "stdio",
  "allowed_origins": [
    "chrome-extension://$extensionId/"
  ]
}
"@ | Out-File -FilePath $manifestPath -Encoding UTF8

Write-Host "Chrome manifest created: $manifestPath" -ForegroundColor Green

# Step 4: Create Edge manifest (optional)
$createEdge = Read-Host "`nCreate Edge manifest too? (y/n)"
if ($createEdge -eq 'y') {
    $edgeDir = "$env:LOCALAPPDATA\Microsoft\Edge\User Data\NativeMessagingHosts"
    New-Item -ItemType Directory -Force -Path $edgeDir | Out-Null
    
    $edgeManifestPath = "$edgeDir\com.anthropic.claude.chrome.json"
    
    @"
{
  "name": "com.anthropic.claude.chrome",
  "description": "Claude in Chrome native messaging host",
  "path": "$batchPathEscaped",
  "type": "stdio",
  "allowed_origins": [
    "chrome-extension://$extensionId/"
  ]
}
"@ | Out-File -FilePath $edgeManifestPath -Encoding UTF8
    
    Write-Host "Edge manifest created: $edgeManifestPath" -ForegroundColor Green
}

# Step 5: Verify WSL setup
Write-Host "`nVerifying WSL setup..." -ForegroundColor Cyan
$wslTest = wsl.exe -e test -f /home/dawson/bin/claude-chrome-host.sh
if ($LASTEXITCODE -eq 0) {
    Write-Host "WSL wrapper script found!" -ForegroundColor Green
} else {
    Write-Host "WARNING: WSL wrapper script not found at /home/dawson/bin/claude-chrome-host.sh" -ForegroundColor Red
    Write-Host "Make sure you completed the WSL setup steps!" -ForegroundColor Yellow
}

# Step 6: Summary
Write-Host "`n========================================" -ForegroundColor Cyan
Write-Host "Setup Complete!" -ForegroundColor Green
Write-Host "========================================" -ForegroundColor Cyan
Write-Host "`nFiles created:"
Write-Host "  - Batch script: $batchPath"
Write-Host "  - Chrome manifest: $manifestPath"
if ($createEdge -eq 'y') {
    Write-Host "  - Edge manifest: $edgeManifestPath"
}

Write-Host "`nNext steps:"
Write-Host "  1. Restart Chrome completely (close all windows)"
Write-Host "  2. Open the Claude in Chrome extension"
Write-Host "  3. Test the connection"
Write-Host "`nIf it doesn't work, check the troubleshooting section in the guide"

Write-Host "`nPress any key to exit..."
$null = $Host.UI.RawUI.ReadKey("NoEcho,IncludeKeyDown")
```

**To run this script:**

1. Save it as `setup-claude-chrome-bridge.ps1`
2. Open PowerShell as Administrator
3. Run:
   ```powershell
   Set-ExecutionPolicy Bypass -Scope Process -Force
   .\setup-claude-chrome-bridge.ps1
   ```

## Security Considerations

<!-- section_id: "20549d01-09ff-47bd-80bc-3a9564880163" -->
### Allowed Origins

The manifest specifies which extensions can connect. Only add trusted extension IDs:

```json
"allowed_origins": [
  "chrome-extension://YOUR_EXTENSION_ID/"
]
```

<!-- section_id: "ac3c4f70-3866-459f-9c25-ea5b2ddbe3c6" -->
### File Permissions

Ensure the batch script and WSL wrapper are only writable by you:

```powershell
# Windows
icacls "$env:USERPROFILE\bin\claude-chrome-host.bat" /inheritance:r /grant:r "$env:USERNAME:F"
```

```bash
# WSL
chmod 700 ~/bin/claude-chrome-host.sh
```

<!-- section_id: "1d20cbe5-6be3-45dd-98da-52fde74f652c" -->
### Logging

Be careful with logging sensitive data. If you enable debug logging, make sure the log file is protected:

```bash
touch ~/claude-chrome-host.log
chmod 600 ~/claude-chrome-host.log
```

To remove the bridge:

**Windows:**
```powershell
# Remove batch script
Remove-Item "$env:USERPROFILE\bin\claude-chrome-host.bat"

# Remove manifests
Remove-Item "$env:LOCALAPPDATA\Google\Chrome\User Data\NativeMessagingHosts\com.anthropic.claude.chrome.json"
Remove-Item "$env:LOCALAPPDATA\Microsoft\Edge\User Data\NativeMessagingHosts\com.anthropic.claude.chrome.json"

# Remove registry keys (if used)
Remove-Item "HKCU:\Software\Google\Chrome\NativeMessagingHosts\com.anthropic.claude.chrome" -Recurse
Remove-Item "HKCU:\Software\Microsoft\Edge\NativeMessagingHosts\com.anthropic.claude.chrome" -Recurse
```

**WSL:**
```bash
# Remove wrapper script
rm ~/bin/claude-chrome-host.sh
rm ~/claude-chrome-host.log  # if logging was enabled
```

## FAQ

<!-- section_id: "5d77b854-cb80-4e67-a099-a2aff0819659" -->
### Q: Can I use this with Firefox?

A: Firefox uses a similar but slightly different Native Messaging protocol. You would need to create a separate manifest in a different location. See [Firefox Native Messaging docs](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/Native_messaging).

<!-- section_id: "5a369523-fb1f-4b2f-a28d-7d6983959ea3" -->
### Q: Does this work with WSL1?

A: It should work, but WSL2 is recommended for better performance and compatibility.

<!-- section_id: "b5af87fd-45df-431d-b430-114eeb723678" -->
### Q: Can I use a different WSL distribution?

A: Yes! Just modify the batch script to specify your distribution:
```batch
wsl.exe -d YourDistroName -e /home/dawson/bin/claude-chrome-host.sh
```

<!-- section_id: "85f2bf47-07df-4fe8-882c-8ad8e4bb5c02" -->
### Q: How do I find my extension ID if I can't see it?

A: 
1. Go to `chrome://extensions/`
2. Enable "Developer mode" (toggle in top-right)
3. Click "Pack extension" (you don't need to actually pack it)
4. The ID will be visible in the extension details

Alternatively, right-click the extension icon → "Manage extension" → check the URL bar for the ID.

<!-- section_id: "ecff24bc-4caf-4dff-bd65-b5ba989c5a33" -->
### Q: Can multiple Chrome profiles use this?

A: Yes! Each Chrome profile will use the same manifest location. Just make sure the extension is installed in each profile.

<!-- section_id: "f9436c9c-7acb-47b0-9adf-d944aa33ab58" -->
### Q: What if my username is different?

A: Replace `/home/dawson` in all scripts with your actual WSL home directory path. You can find it by running `echo $HOME` in WSL.

<!-- section_id: "7bcc96e5-ef60-4813-bf9e-0441715e60e0" -->
### Q: Does this work with Brave or other Chromium browsers?

A: Yes! Chromium-based browsers use the same Native Messaging protocol. You'll need to create a manifest in the appropriate location for each browser.
