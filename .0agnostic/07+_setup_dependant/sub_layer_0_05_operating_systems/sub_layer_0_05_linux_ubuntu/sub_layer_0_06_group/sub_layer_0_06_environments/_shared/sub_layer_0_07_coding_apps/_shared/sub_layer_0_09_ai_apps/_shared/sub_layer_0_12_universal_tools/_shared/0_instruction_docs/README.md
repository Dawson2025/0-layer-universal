# Universal Tools
*Trickle-Down Level 0.75: Universal Tools and Utilities*

<!-- section_id: "305d6dbf-7b4f-4c71-9ab2-77228307dc3e" -->
## Overview

The Universal Tools section contains tools, utilities, and implementations that can be applied across any project or technology stack. These tools are more concrete than the universal instructions in Level 0, but more general than the project-specific tools in Level 1.5.

<!-- section_id: "a16e6be0-2620-4de1-ae30-fe935a329fa0" -->

**Location**: `mcp-tools/`

- **Automated Deployment**: Scripts for deployment and management
- **Health Monitoring**: Built-in health checks and status monitoring

- **Search & Research**: Web search, GitHub search, Context7 documentation
- **System Integration**: Filesystem, Slack, PostgreSQL
- **Documentation**: Context7 local and remote servers

#### 5.3 Configuration Management
- **Environment-Specific**: Development, production, and testing configs
- **API Key Management**: Secure handling of API credentials
- **Easy Switching**: Simple commands to change configurations
- **Backup & Recovery**: Automatic configuration backups

### 6. AI Development Frameworks
**Location**: `ai-development-frameworks/`
**Purpose**: Comprehensive frameworks for structured AI-driven development workflows

#### 6.1 Spec-Driven Development (Spec Kit)
- **GitHub Spec Kit**: Structured 4-phase validation process
- **Version-Controlled Specifications**: Specifications alongside code
- **AI Integration**: Works with Copilot, Claude Code, Gemini CLI
- **Quality Assurance**: Validation checkpoints at each phase

#### 6.2 Agentic Team Development (BMAD Method)
- **Specialized AI Agents**: Analyst, Product Manager, Architect, Scrum Master, Product Owner, Developer, QA
- **Git-Based Artifact Versioning**: Complete audit trail from ideation to implementation
- **Context-Engineered Development**: Hyper-detailed development stories
- **Collaborative Review**: Human + AI review processes

#### 6.3 AI Coding Assistants
- **Cursor**: AI-first code editor with deep codebase context
- **Windsurf**: Real-time collaboration with massive context windows
- **GitHub Copilot**: Industry-standard AI pair programmer
- **Aider**: Terminal-based pair programmer with git awareness
- **Claude Code**: Deep reasoning and architectural insights
- **Qwen3-Coder**: Privacy-focused local coding model

#### 6.4 Workflow Optimization
- **Phase-Based Tool Selection**: Choose right tool for each development phase
- **Workflow Patterns**: Optimized patterns for different team sizes
- **Integration Strategies**: Combining frameworks with assistants
- **Decision Frameworks**: Systematic tool selection methodology

#### 6.5 Integration
- **Framework Comparison**: Detailed comparison of Spec Kit vs BMAD
- **Tool Integration**: Integration with Cursor, Codex, Claude Code
- **Combined Workflows**: Using both frameworks together for maximum coverage

### 7. Platform Version Control System
**Location**: `platform-version-control/`
**Purpose**: Comprehensive version control for all app development platforms and services

#### 7.1 Core Platforms
- **Cloud Providers**: AWS, GCP, Azure, DigitalOcean
- **Hosting**: Vercel, Netlify, Railway, Render, Fly.io, Heroku
- **Authentication**: Auth0, Clerk, Supabase Auth, Firebase Auth
- **Storage & CDN**: S3, Cloud Storage, Cloudinary, Cloudflare
- **Serverless**: Lambda, Cloud Functions, Edge Functions
- **Services**: Email, SMS, Payments, Analytics, Monitoring

#### 7.2 Universal Principles
- **Infrastructure as Code**: Terraform, Pulumi, CloudFormation
- **Configuration as Code**: Version all platform configs
- **Environment Parity**: Dev → Staging → Production
- **Secret Management**: Secure credential handling
- **CI/CD Integration**: Automated deployments

#### 7.3 Database Integration
- **Databases**: See `platform-version-control/databases/` subdirectory
- **Migration Tools**: Liquibase, Flyway, Supabase CLI, Firebase CLI
- **Coordination**: Database + platform deployment workflows
- **Schema Management**: Version-controlled database changes

#### 7.4 Comprehensive Coverage
- **Complete Platform Stack**: Every service your app uses
- **Universal Practices**: Applicable to all platforms
- **Platform-Specific Guides**: Detailed implementation for each
- **Best Practices**: Proven patterns and workflows
