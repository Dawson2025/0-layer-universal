## API Key Configuration Issues

### Error: "Authentication failed" or "Invalid API key"

**Symptoms:**
- Context7 tools return authentication errors
- Error messages mentioning "unauthorized" or "invalid credentials"

**Causes:**
1. Missing `CONTEXT7_API_KEY` environment variable
2. Incorrect or expired API key
3. API key not properly passed to the MCP server process

**Solutions:**

1. **Verify the API key is set:**
   ```bash
   echo $CONTEXT7_API_KEY
   ```

2. **Check MCP configuration:**
   ```json
   {
     "mcpServers": {
       "context7": {
         "env": {
           "CONTEXT7_API_KEY": "your-actual-key-here"
         }
       }
     }
   }
   ```

3. **Regenerate API key:**
   - Visit [context7.io](https://context7.io)
   - Navigate to API Keys section
   - Generate a new key and update your configuration

4. **Use wrapper scripts:**
   The wrapper scripts generated by `mcp_manager.py` ensure environment variables are properly set:
   ```bash
   python3 setup/scripts/mcp_manager.py --scope user
   ```

### Error: "API key format invalid"

**Symptoms:**
- Server starts but immediately fails on first request
- Error message about key format

**Solutions:**

1. **Check for extra whitespace:**
   ```bash
   # Trim whitespace from key
   export CONTEXT7_API_KEY="$(echo -n "$CONTEXT7_API_KEY" | tr -d '[:space:]')"
   ```

2. **Verify key format:**
   Context7 API keys are typically UUID format: `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

3. **Check for quote issues in config files:**
   Ensure proper quoting in JSON:
   ```json
   "CONTEXT7_API_KEY": "key-here"  // Correct
   "CONTEXT7_API_KEY": key-here    // Wrong - missing quotes
   ```

## Connection Timeouts

### Error: "Connection timeout" or "ETIMEDOUT"

**Symptoms:**
- Requests hang and eventually fail
- Long delays before error responses
- Intermittent failures

**Causes:**
1. Network connectivity issues
2. Firewall blocking outbound connections
3. DNS resolution problems
4. Context7 API service issues

**Solutions:**

1. **Test connectivity:**
   ```bash
   # Test DNS resolution
   nslookup api.context7.io

   # Test HTTPS connectivity
   curl -v https://api.context7.io/health
   ```

2. **Check firewall rules:**
   ```bash
   # Ensure outbound HTTPS (port 443) is allowed
   sudo ufw status
   ```

3. **Use a proxy if required:**
   ```bash
   export HTTPS_PROXY="http://proxy.example.com:8080"
   ```

4. **Check Context7 service status:**
   - Visit Context7 status page
   - Check their official communication channels

### Error: "Request timeout" during large documentation requests

**Symptoms:**
- Small requests work, large documentation retrieval fails
- Timeout occurs mid-request

**Solutions:**

1. **Reduce token count:**
   ```
   Use smaller token limits in get-library-docs:
   { "tokens": 5000 }  // Instead of default 10000
   ```

2. **Use topic filtering:**
   ```
   Specify a topic to narrow the documentation scope:
   { "topic": "hooks" }  // Instead of all documentation
   ```

3. **Retry with exponential backoff:**
   If using programmatically, implement retry logic with increasing delays.

## Rate Limiting Problems

### Error: "Rate limit exceeded" or "429 Too Many Requests"

**Symptoms:**
- Requests fail with 429 status code
- Error message about rate limits
- Requests work initially then start failing

**Causes:**
1. Exceeding API request quota
2. Too many concurrent requests
3. Burst requests triggering rate limits

**Solutions:**

1. **Implement request throttling:**
   Space out your requests. Wait at least 1-2 seconds between documentation requests.

2. **Check your API plan limits:**
   - Log into Context7 dashboard
   - Review your current usage and limits
   - Consider upgrading if hitting limits regularly

3. **Cache responses locally:**
   For frequently accessed documentation, consider caching responses to reduce API calls.

4. **Use batch requests wisely:**
   If fetching multiple library docs, space them out rather than requesting all at once.

### Error: "Daily quota exceeded"

**Symptoms:**
- All requests fail with quota errors
- Error mentions daily limits

**Solutions:**

1. **Wait for quota reset:**
   Daily quotas typically reset at midnight UTC.

2. **Upgrade your API plan:**
   Contact Context7 for higher quota tiers.

3. **Optimize API usage:**
   - Cache responses
   - Request only needed topics
   - Use smaller token limits

## MCP Server Startup Failures

### Error: "Cannot find module @upstash/context7-mcp"

**Symptoms:**
- Server fails to start
- NPX cannot find the package

**Solutions:**

1. **Clear NPX cache:**
   ```bash
   rm -rf ~/.npm/_npx
   ```

2. **Verify npm registry access:**
   ```bash
   npm ping
   npm search @upstash/context7-mcp
   ```

3. **Use explicit version:**
   ```json
   "args": ["-y", "@upstash/context7-mcp@latest"]
   ```

### Error: "npx: command not found"

**Symptoms:**
- MCP server doesn't start
- Error about missing npx command

**Solutions:**

1. **Check Node.js installation:**
   ```bash
   which node
   which npx
   node --version
   ```

2. **Source NVM if using:**
   ```bash
   source ~/.nvm/nvm.sh
   nvm use default
   ```

3. **Use wrapper scripts:**
   The `mcp_manager.py` wrapper scripts explicitly set `PATH` to include Node.js:
   ```bash
   python3 setup/scripts/mcp_manager.py --scope user
   ```

### Error: MCP server starts but tools are not available

**Symptoms:**
- No error messages
- Context7 tools not appearing in tool list

**Solutions:**

1. **Restart AI tool:**
   After configuration changes, restart your AI CLI (Claude Code, Codex, etc.).

2. **Check configuration syntax:**
   Validate your JSON configuration:
   ```bash
   python3 -m json.tool ~/.config/mcp/mcp.json
   ```

3. **Verify server is running:**
   Check for npx processes:
   ```bash
   ps aux | grep context7
   ```

## Tool Availability Issues

### Error: "Tool 'resolve-library-id' not found"

**Symptoms:**
- Server starts but tools are unavailable
- Other MCP servers work, Context7 doesn't

**Solutions:**

1. **Check MCP configuration:**
   Ensure `context7` is properly defined in your MCP config.

2. **Restart after configuration changes:**
   MCP configurations are loaded at startup. Restart your AI tool.

3. **Check for competing configurations:**
   Ensure no duplicate `context7` entries in different config files.

### Error: "Library not found" when using resolve-library-id

**Symptoms:**
- Search returns no results
- Common libraries not found

**Solutions:**

1. **Try different search terms:**
   - Use the full package name: `react` vs `reactjs`
   - Try the npm package name: `@types/react`
   - Include organization: `facebook/react`

2. **Check Context7 library coverage:**
   Not all libraries are indexed. Check Context7 documentation for supported libraries.

## Environment Variable Problems

### Error: Environment variables not passed to MCP server

**Symptoms:**
- Variables set in shell but not seen by MCP server
- Server starts but authentication fails

**Causes:**
MCP servers run via `npx` in isolated environments and don't inherit shell variables.

**Solutions:**

1. **Set variables in MCP config:**
   ```json
   {
     "mcpServers": {
       "context7": {
         "env": {
           "CONTEXT7_API_KEY": "your-key",
           "HOME": "/home/youruser"
         }
       }
     }
   }
   ```

2. **Use wrapper scripts:**
   Wrapper scripts from `mcp_manager.py` handle environment properly:
   ```bash
   python3 setup/scripts/mcp_manager.py --scope user
   ```

3. **Check wrapper script contents:**
   Verify the generated wrapper includes necessary exports:
   ```bash
   cat ~/.config/mcp/servers/mcp-context7-generic.sh
   ```

### Error: HOME or PATH not set correctly

**Symptoms:**
- NPX fails to find packages
- Configuration files not found

**Solutions:**

1. **Explicitly set in config:**
   ```json
   "env": {
     "HOME": "/home/dawson",
     "PATH": "/home/dawson/.nvm/versions/node/v20.0.0/bin:/usr/local/bin:/usr/bin:/bin"
   }
   ```

2. **Run mcp_manager.py to auto-detect:**
   The script automatically detects and configures these paths:
   ```bash
   python3 setup/scripts/mcp_manager.py --scope user
   ```

## Diagnostic Commands

Use these commands to diagnose Context7 MCP issues:

```bash
# Check if Context7 package is accessible
npx -y @upstash/context7-mcp --help

# Verify environment variables
env | grep CONTEXT7

# Test API connectivity
curl -H "Authorization: Bearer $CONTEXT7_API_KEY" https://api.context7.io/health

# Check MCP configuration
cat ~/.config/mcp/mcp.json | python3 -m json.tool

# View running MCP processes
ps aux | grep -E "(npx|context7|mcp)"

# Check Node.js/NPX path
which npx
npx --version

# View MCP server logs (if available)
journalctl -u mcp-context7 --no-pager -n 50
```
