### Cursor Agent Context Delivery

Cursor Agent reads context from (in priority order):
1. **Team Rules** — Org-wide dashboard rules (Team/Enterprise plans)
2. **Project Rules** — `.cursor/rules/*.mdc` files with YAML frontmatter
3. **User Rules** — Global Cursor settings
4. **AGENTS.md** — Plain markdown at repo root (generated by agnostic-sync.sh)

The layer-stage system delivers context via `agnostic-sync.sh`:
- Currently: generates `.cursorrules` (deprecated, lean format)
- Future: generate `.cursor/rules/*.mdc` (modern format with frontmatter)
- Also: generates `AGENTS.md` (full static content, read by Cursor)
