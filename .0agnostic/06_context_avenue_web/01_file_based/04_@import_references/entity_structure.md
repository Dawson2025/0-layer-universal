# Canonical Entity Structure

This is the **single source of truth** for entity directory structure in the layer-stage system. All entity creation tools, skills, guides, and constraints reference this file.

## Full Entity Structure

Every entity (project, feature, component, research project) follows this canonical structure. `N` is the entity's own layer number; `N+1` is the child layer.

```

├── README.md                              # Human-readable overview

├── AGENTS.md                              # Auto-generated by agnostic-sync.sh
├── GEMINI.md                              # Auto-generated by agnostic-sync.sh
```

## Internal Directories Also Need 0AGNOSTIC.md

Every directory inside `layer_N_group/` should have its own `0AGNOSTIC.md` + auto-generated tool files:

- `layer_N_group/layer_N_00_layer_registry/0AGNOSTIC.md`
- `layer_N_group/layer_N_99_stages/0AGNOSTIC.md` (and each stage_N_XX/)

Run `agnostic-sync.sh` on each `0AGNOSTIC.md` to generate CLAUDE.md, AGENTS.md, GEMINI.md, OPENAI.md.

## Agent Files

AALang agent files live inside `.0agnostic/06_context_avenue_web/01_file_based/01_aalang/` at each level (entity root, stages).

**Naming conventions**:
- `{layer/stage}.orchestrator.gab.jsonld` — Orchestrator (e.g., `layer_1.orchestrator.gab.jsonld`, `stage_01.orchestrator.gab.jsonld`)
- `{purpose}.gab.jsonld` — Full GAB agent graph (e.g., `agent_delegation_system.gab.jsonld`)
- `{purpose}.agent.jsonld` — Lightweight stub (e.g., `stage_delegator.agent.jsonld`)

**Integration summaries**: Each `.gab.jsonld` should have a matching `.integration.md` in `06_context_avenue_web/02_aalang_markdown_integration/`.

**Generate**: `bash .0agnostic/01_knowledge/layer_stage_system/resources/tools/jsonld-to-md.sh <path-to-jsonld>`

## Naming Conventions

| Convention | Rule | Example |
|---|---|---|
| **Internal layer dir** | MUST use `_group` suffix | `layer_7_group/` (NOT `layer_7/`) |
| **Children dir** | MUST use `_group` suffix | `layer_8_group/` (NOT `layer_8/`) |
| **Episodic memory** | MUST be named `episodic_memory` | `.0agnostic/04_episodic_memory/` (NOT `episodic/`) |
| **Agent files** | Use dot-separation | `name.gab.jsonld` (NOT `name_gab.jsonld`) |
| **AALang location** | Always in context avenue web | `.0agnostic/06_context_avenue_web/01_file_based/01_aalang/` |
| **Handoff documents** | Always in .0agnostic | `.0agnostic/05_handoff_documents/` (NOT in layer_N_group/) |
| **Entity naming** | `layer_{N}_{type}_{name}` | `layer_2_feature_assignments` |
| **.0agnostic/ subdirs** | All numbered | `01_knowledge/`, `02_rules/`, etc. |
| **Knowledge topics** | Self-contained per topic | `01_knowledge/[topic]/{principles,docs,resources}` |

## Stage Creation Template

Create ALL 12 stages (00-11). **Empty stages are valid. Missing stages are NOT.**

```bash
# Create stage 00 (registry)
mkdir -p "layer_N_group/layer_N_99_stages/stage_N_00_stage_registry/"{outputs,synthesis}

# Create stages 01-11 (each gets outputs/ and synthesis/)
for i in 01_request_gathering 02_research 03_instructions 04_design 05_planning 06_development 07_testing 08_criticism 09_fixing 10_current_product 11_archives; do
  mkdir -p "layer_N_group/layer_N_99_stages/stage_N_$i/"{outputs,synthesis}
done

# Add config directories to EACH stage (numbered .0agnostic/ — same as entity root)
for stage_dir in layer_N_group/layer_N_99_stages/stage_N_*/; do
  mkdir -p "$stage_dir"/{.0agnostic/{01_knowledge,02_rules/{static,dynamic},03_protocols,04_episodic_memory/{sessions,changes},05_handoff_documents/{01_incoming/{01_from_above,02_from_sides/{01_from_left,02_from_right},03_from_below},02_outgoing/{01_to_above,02_to_sides/{01_to_left,02_to_right},03_to_below}},06_context_avenue_web/{00_context_avenue_web_registry,01_file_based/{01_aalang,02_aalang_markdown_integration,03_auto_memory,"04_@import_references",05_skills,06_agents,07_path_specific_rules,08_hooks},02_data_based/{09_knowledge_graph,10_relational_index,11_vector_embeddings,12_temporal_index,13_shimi_structures}},07+_setup_dependant},.1merge/{.1claude_merge/{0_synced,1_overrides,2_additions},.1cursor_merge/{0_synced,1_overrides,2_additions},.1gemini_merge/{0_synced,1_overrides,2_additions},.1aider_merge/{0_synced,1_overrides,2_additions},.1codex_merge/{0_synced,1_overrides,2_additions},.1copilot_merge/{0_synced,1_overrides,2_additions}},.claude/{rules,episodic_memory/{sessions,changes}},.cursor/{rules,episodic_memory/{sessions,changes}},.gemini/episodic_memory/{sessions,changes},.codex/episodic_memory/{sessions,changes},.github/instructions,synthesis}
```

## Required Files

After creating the directory structure, create these files:

| File | Location | Purpose | How |
|------|----------|---------|-----|
| `0AGNOSTIC.md` | Entity root | Identity, triggers, pointers | Write manually (see INSTANTIATION_GUIDE.md) |
| `0INDEX.md` | Entity root | Navigation index | Write manually |
| `README.md` | Entity root | Human-readable overview | Write manually |
| `CLAUDE.md` etc. | Entity root | Tool-specific context | Run `agnostic-sync.sh` |
| `0AGNOSTIC.md` | Every internal dir | Directory-level context | Write manually, run agnostic-sync.sh |
| `layer_N.orchestrator.gab.jsonld` | `.0agnostic/06_context_avenue_web/01_file_based/01_aalang/` | Entity orchestrator | Copy from sibling, adapt |
| `{purpose}.gab.jsonld` | `.0agnostic/06_context_avenue_web/01_file_based/01_aalang/` | Full GAB agent | Create per entity |
| `{purpose}.agent.jsonld` | `.0agnostic/06_context_avenue_web/01_file_based/01_aalang/` | Lightweight stub | Create per entity |
| `layer_N_99_stages.orchestrator.gab.jsonld` | `layer_N_99_stages/` | Stages orchestrator | Create or copy from sibling |
| `stage_XX.orchestrator.gab.jsonld` | Each stage's `.0agnostic/06_context_avenue_web/01_file_based/01_aalang/` | Stage orchestrator | Create per stage |
| `status_N.json` | `layer_N_99_stages/` | Stage workflow tracker | Create with stage status |

## Key Conventions

- Internal content uses `layer_N_group/` with the **_group suffix** — NOT bare `layer_N/`
- Stages go inside `layer_N_group/layer_N_99_stages/`, NOT at entity root
- **Entity-scoped resources live in `.0agnostic/`** with numbered subdirectories (01-07+)
- Children go in `layer_N+1_group/`, with the child's own layer number = N+1
- `N` always matches the entity's own layer: project=1, feature=2, research feature=0, etc.
- Knowledge is organized per-topic: each topic in `01_knowledge/` has `principles/`, `docs/`, `resources/{templates,tools/scripts}`
- Rules always has `static/` (always-on constraints) and `dynamic/` (trigger-based with protocol pointers) subdirectories
- Episodic memory is always named `episodic_memory/` (NOT `episodic/`)
- `.0agnostic/04_episodic_memory/` is the source of truth — tool dirs receive synced copies
- Handoff documents live in `.0agnostic/05_handoff_documents/` (NOT in `layer_N_group/`)
- Context avenues live in `.0agnostic/06_context_avenue_web/` with two subdirs: `01_file_based/` (01-08) and `02_data_based/` (09-13)
- `outputs/` and `synthesis/` live inside each **stage**, NOT at entity root
- Every tool dir (`.claude/`, `.cursor/`, `.gemini/`, `.codex/`) gets its own `episodic_memory/`
