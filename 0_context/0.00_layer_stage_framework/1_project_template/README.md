# Project Layer Template (1.x)

Use this to scaffold any project-level context. Mirrors the universal 0.x stack but specialized for a single project.

## Slots
- 1.00_basic_prompts: project init + what-to-do-next prompts.
- 1.10_project_se_knowledge: SE/domain knowledge this project relies on.
- 1.20_project_principles: project-specific design/UX/security principles.
- 1.30_project_rules: hard rules (branching, testing, compliance).
- 1.40_project_os: OS targets/requirements for dev/prod.
- 1.45_project_env_repo_layout: local/remote env details; repo layout/monorepo notes.
- 1.50_project_architecture: stack (FE/BE/Data/Auth/Storage/Integrations) + diagrams.
- 1.55_project_coding_app_setup: IDE/run/debug configs for this project.
- 1.60_project_apps_browsers_tools: dashboards, admin panels, monitoring tools.
- 1.70_project_ai_app_tool_usage: approved AI tools/agents and scopes.
- 1.80_project_model_usage: approved models for this project and their uses.
- 1.90_project_tools: project-specific scripts/CLIs/migrations.
- 1.99_stages: stage folders and status template.

## Stages (1.99)
0.0 instructions → 0.1 planning → 0.2 design → 0.3 development → 0.4 testing → 0.5 criticism → 0.6 fixing → 0.7 archives.

Copy, rename to your project, and populate each slot.
