# Stage 09: Archives

## Purpose
Store and maintain historical records of previous versions, iterations, and project artifacts. This stage provides a reference for past decisions, enables rollback capabilities, and preserves institutional knowledge.

## Entry Criteria
- Version superseded in Stage 08
- Or project/iteration completed
- Artifacts identified for archival
- Archive location prepared

## Exit Criteria
- Artifacts properly archived
- Archive indexed and searchable
- Retention policy applied
- Access controls set
- Archive integrity verified

## Typical Tasks
- Identify artifacts for archival
- Organize archive structure
- Transfer artifacts to archive
- Create archive index/manifest
- Apply retention policies
- Set access permissions
- Verify archive integrity
- Document archive contents
- Maintain archive metadata

## Handoffs
- **From Previous**: Superseded versions, completed project artifacts
- **To Next**: Archived materials available for reference

## Directory Structure
Each stage directory contains:
- `CLAUDE.md` - Stage-specific context
- `ai_agent_system/` - AI tool configurations
- `hand_off_documents/` - Handoff files
- `docs/` - Stage documentation
- `work/` - Working files

## Key Artifacts
- Archived versions
- Historical documentation
- Decision records
- Project retrospectives
- Archive manifest/index
- Retention schedules

## Archive Organization
Archives should be organized by:
- Project/feature name
- Version number
- Date archived
- Archive type (code, docs, artifacts)

## Common Pitfalls
- Unorganized archives
- Missing metadata
- No retention policy
- Poor searchability
- Incomplete archives
- Not verifying integrity
