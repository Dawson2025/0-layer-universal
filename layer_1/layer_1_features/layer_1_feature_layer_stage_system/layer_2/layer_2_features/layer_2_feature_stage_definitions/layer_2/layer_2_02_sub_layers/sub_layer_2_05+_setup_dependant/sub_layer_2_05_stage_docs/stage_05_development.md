# Stage 04: Development

<!-- section_id: "14328248-c58c-4894-979f-1047f4d358a1" -->
## Purpose
Implement the designed solution by writing code, creating content, or building the actual deliverable. This stage transforms designs into working products through iterative development and continuous integration.

<!-- section_id: "70e7ef5c-365d-4c63-9612-fc1105a23958" -->
## Entry Criteria
- Design documents completed from Stage 03
- Development environment set up
- Access to required tools and resources
- Coding standards and guidelines available
- Version control initialized

<!-- section_id: "81ab82b1-73e3-4a05-97ad-1cb9ddbbf274" -->
## Exit Criteria
- All planned features implemented
- Code follows established standards
- Unit tests written and passing
- Code reviewed and approved
- Documentation updated
- Build successful with no critical errors
- Ready for testing phase

<!-- section_id: "6c512cde-7b43-496a-942e-8083dbe2deab" -->
## Typical Tasks
- Set up development environment
- Implement features according to design
- Write unit tests
- Conduct code reviews
- Refactor for quality
- Update documentation
- Integrate with existing systems
- Handle error cases
- Optimize performance

<!-- section_id: "c7fe82e4-e142-4fd9-8c7b-77663d604b61" -->
## Handoffs
- **From Previous**: Architecture documents, component designs, interface specs
- **To Next**: Working implementation, unit tests, updated documentation, build artifacts

<!-- section_id: "7470c8dc-d9fc-4d9c-94b8-e8ef519a9b2c" -->
## Directory Structure
Each stage directory contains:
- `CLAUDE.md` - Stage-specific context
- `ai_agent_system/` - AI tool configurations
- `hand_off_documents/` - Handoff files
- `docs/` - Stage documentation
- `work/` - Working files

<!-- section_id: "9303203d-8480-4757-b0bb-129836834985" -->
## Key Artifacts
- Source code
- Unit tests
- Build scripts
- Development documentation
- Code review records
- Integration logs

<!-- section_id: "235ea7fb-f1c9-47bc-b7e4-f80c6cc5f96c" -->
## Common Pitfalls
- Deviating from design without approval
- Skipping unit tests
- Poor code quality
- Insufficient documentation
- Not following coding standards
- Ignoring technical debt
